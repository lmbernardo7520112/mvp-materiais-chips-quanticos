# ADR-004 — v0.2 Scope Selection

## Status
Accepted

> **Acceptance Record — 2026-05-06**
>
> ADR-004 is formally promoted from `Proposed` to `Accepted`.
>
> - The v0.2 specification was merged into `main` via [PR #2](https://github.com/lmbernardo7520112/mvp-materiais-chips-quanticos/pull/2).
> - The v0.2 scope is approved as a **numerical robustness milestone**.
> - **MUST core:** Domain2D, thermal solver 2D, 2D stability guard, convergence analysis (manufactured analytical solution), pytest-cov ≥70% (blocking).
> - **SHOULD conditional:** Diffusion 2D (requires all MUST gates green first).
> - **WON'T:** Poisson, Schrödinger, TCAD/QTCAD, Czochralski, phase-field, quantum coherence, defect-like reaction-diffusion.
> - C remains an **adimensional proxy** — not physical concentration.
> - Implementation requires its own feature branch and controlled execution per the approved plan.

## Context

A versão v0.1 do projeto `mvp-materiais-chips-quanticos` foi concluída com sucesso:
- 21/21 testes passando.
- ruff verde.
- CI verde (matrix Python 3.11+3.12).
- Tag v0.1.0 em main.
- PR #1 mergeado.
- Governança completa (ADRs, walkthrough, audit, debt scorecard).

A ADR-003 deferiu o caso 2D para v0.2.

A ADR-005 estabeleceu o roadmap process-to-device em ondas:
- v0.2: robustez numérica
- v0.3: defect-like reaction-diffusion
- v0.4: eletrostática base (Poisson)
- v0.5: confinamento quântico (Schrödinger)
- v1.0: demonstrador completo

A deliberação do Research Council (branch `research/process-to-device-bridge-deliberation`)
confirmou que v0.2 deve focar em robustez numérica e extensão 2D, sem introduzir
física nova pesada.

Candidatos para v0.2 incluem:
1. Domain2D e solver térmico 2D.
2. Solver difusivo 2D.
3. Análise de convergência.
4. pytest-cov.
5. Notebooks/Jupytext.
6. Sensibilidade global (Morris/Sobol).
7. Poisson 2D.
8. Defect-like reaction-diffusion com G(T) e R(T,C).

## Decision

A v0.2 é definida como uma etapa de **robustez numérica e extensão 2D controlada**,
com a seguinte priorização:

### MUST (obrigatório)
- Domain2D estruturado (grade retangular uniforme).
- Solver térmico 2D explícito (Euler + diferenças finitas centradas).
- Critério de estabilidade 2D com guard que recusa dt instável.
- Testes de shape, finitude, estabilidade, regressão.
- Análise de convergência com caso analítico manufaturado (CSV + figura).
- pytest-cov com meta ≥70% — **CI falha abaixo da meta** (`--cov-fail-under=70`).
- Documentação de limitações v0.2.
- Não regressão completa da v0.1.
- ADR-005 e evolution_deliberation.md presentes na branch.
- Política additive-only para APIs 1D.

### SHOULD (desejável)
- Solver difusivo 2D (Arrhenius + Neumann no-flux). **Condicional:** só pode ser
  implementado após todos os MUST gates verdes. `generate_all_results.py` NÃO
  depende de difusão 2D. Se deferido, registrar em TD.
- Figuras 2D (contour plots) geradas por CLI.
- Jupytext ou notebooks demonstrativos.
- Relatório técnico v0.2.

### COULD (opcional)
- Sensibilidade global por Morris screening.
- Release notes institucionais.
- Resumo executivo para CNPEM/LNNano/INCT-DQ.

### WON'T (explicitamente fora do escopo)
- Poisson 2D (requer ρ_eff de v0.3).
- Schrödinger (requer potencial de v0.4).
- TCAD/QTCAD (quebra ADR-001).
- Czochralski real (tese de doutorado).
- Phase-field (fora da rota ADR-005).
- OpenFOAM/FEniCSx/MOOSE/PRISMS-PF (quebra ADR-001).
- Coerência quântica (ADR-002 permanente).
- Validação experimental.
- Defect-like reaction-diffusion completo (G(T), R(T,C) — escopo v0.3).
- Parâmetros reais calibrados sem documentação.
- Transformar C em concentração física.

## Rationale

1. **Infraestrutura primeiro:** v0.3+ requer Domain2D e solvers 2D validados.
   Sem eles, novas equações serão construídas sobre base frágil.

2. **Convergência:** sem análise de convergência dx/dt, não há como demonstrar
   que os resultados numéricos são confiáveis.

3. **Coverage:** sem pytest-cov, não há como medir o impacto de novos módulos
   na qualidade de teste.

4. **Separação de módulos:** solvers 2D são arquivos separados para evitar
   regressão nos 1D existentes.

5. **ADR-005 compliance:** v0.2 = "robustez numérica e convergência" conforme
   roadmap aceito.

6. **Deliberação do Research Council:** "v0.2 não deve conter física nova,
   mas apenas focar em blindar numericamente a base."

## Consequences

### Positivas
- Base 2D sólida para v0.3+.
- Convergência documentada antes de adicionar equações acopladas.
- Coverage medido e reportado.
- Regressão zero garantida por design (módulos separados).
- Alinhamento com ADR-005 e deliberação.

### Negativas
- Escopo visualmente menos "impressionante" (ainda demonstrativo).
- Difusão 2D como SHOULD pode ser deferida.
- Sem avanço na direção process-to-device além da infraestrutura.

## Alternatives Considered

1. **Implementar Poisson já na v0.2.**
   - REJEITADO. Poisson requer ρ_eff, que requer C_def (v0.3). Pular essa
     etapa criaria acoplamento frágil e sem testes de base.

2. **Implementar defect-like reaction-diffusion na v0.2.**
   - REJEITADO (ou deferido). G(T) e R(T,C) requerem parametrização
     teórica que ainda não foi curada. Domain2D precisa existir primeiro.

3. **Fazer apenas notebooks, sem 2D.**
   - REJEITADO. Notebooks sem 2D não avançam a infraestrutura numérica.
     O roadmap ADR-005 é claro: v0.2 = robustez numérica.

4. **Focar em robustez numérica + 2D térmico como core.**
   - ACEITO. Esta opção maximiza infraestrutura com risco mínimo.

## Guardrails

- C segue como proxy adimensional.
- Toda física nova (além do já planejado em MUST/SHOULD) exige nova ADR.
- Nenhuma alegação de wafer real.
- Nenhuma predição de coerência quântica.
- Figuras 2D devem conter disclaimer "[demonstrativo — não calibrado]".
- Toda iteração v0.2 herda e passa na suíte completa de regressão v0.1.
- Solvers 1D existentes não são modificados.
- **Política additive-only:** `thermal_solver.py` e `diffusion_solver.py` intocados.
  `domain.py` e `config.py` só estendidos. `plots.py` e `generate_all_results.py`
  só estendidos se artefatos v0.1 continuarem sendo gerados.
- **Coverage bloqueante:** CI falha se `--cov-fail-under=70` não for atingido.
- **Convergência:** caso analítico manufaturado obrigatório (equação do calor 2D Dirichlet homogêneo).
- **Difusão 2D condicional:** só após MUST gates verdes.
