## Context

The v0.3.5 release completed parameter curation and documented three
candidate mappings from C_def to ρ_eff (Options A, B, C). A fourth option,
**Option C-B** (Boussinesq-inspired linearized trap-charge closure), was
subsequently proposed and evaluated by the research council.

The current state:
- C_def is a **dimensionless proxy** (tier TX, adimensional).
- No Poisson solver exists.
- No ρ_eff field exists.
- The general Poisson form ∇·(ε∇φ) = −ρ_eff is documented.
- ∇²φ = −ρ_eff/ε is accepted only as a homogeneous-permittivity simplification.
- Option C-B is the council-recommended candidate.

**The problem:** The MVP needs a principled, testable, and overclaim-resistant
strategy to bridge from process-level fields (T, C_def) to an electrostatic
source term (δρ_eff) that can drive a Poisson solver, without claiming
physical accuracy.

## Decision

Because ADR-007 is now Accepted, it approves the scoped v0.4 planning direction only. A future implementation PR may implement the following items only after policy activation, updated quality gates, tests-first planning, and CI approval:

### 1. Charge Closure — Option C-B

A Boussinesq-inspired linearized trap-charge closure:

```
θ(x,y) = [T(x,y) − T_ref] / ΔT_ref

h(C_def) = C_def / C_sat

D_it_eff(x,y) = D_it_ref · [1 + λT·θ + λC·h(C_def)]

ρ_eff_raw(x,y) = q · D_it_eff · I(T)

δρ_eff(x,y) = ρ_eff_raw − ⟨ρ_eff_raw⟩_spatial
```

### 2. Poisson Solver — 2D Demonstrative

```
∇·(ε∇φ) = −δρ_eff
```

With homogeneous ε (simplified to ∇²φ = −δρ_eff/ε), Dirichlet or
Neumann boundary conditions, and explicit reference/background charge definition via mean
subtraction of δρ_eff.

### 3. Conceptual Limitations

#### Boussinesq analogy: perturbation principle, not physical equivalence
The Boussinesq analogy is used here as a modeling principle: small deviations from a reference/background material state are converted into an effective source term for a governing equation. It does not imply that semiconductor trap-charge response is physically equivalent to fluid-density buoyancy, nor that the response is generally linear. The useful transfer is the perturbation/background logic, not the mass-density physics.

#### Small-perturbation validity
The C-B closure is valid only as a small-perturbation MVP approximation. The terms λTθ and λC h(C_def) are not assumed to approximate the full nonlinear semiconductor charge response. They are demonstrative sensitivities used to generate a controlled perturbation source. Large-amplitude perturbations would require nonlinear occupancy and self-consistent Poisson treatment.

#### Self-consistency limitation
In real semiconductor electrostatics, trap occupancy is not independent of the electrostatic potential. A physically complete model would require a self-consistent loop φ → trap occupancy → rho_eff → φ, similar in spirit to nonlinear Poisson/Poisson-Boltzmann formulations. The v0.4 MVP deliberately uses a static decoupled δρ_eff source to test the process-to-electrostatics bridge. This is a numerical/scientific scaffold, not a predictive semiconductor electrostatics model.

### 4. Parameter Classification

| Parameter | Tier | Status |
|-----------|------|--------|
| D_it_ref | T3 | uncalibrated demonstrative numerical scale / calibration-required |
| N_ref | T0 or T3 | uncalibrated demonstrative numerical scale / calibration-required |
| λT | T0 | uncalibrated sensitivity parameter |
| λC | T0 | uncalibrated sensitivity parameter |
| ε_Si | T2 | well-known: 11.7 ε₀ |
| q | CONST | physical constant |
| T_ref | T0 | spatial mean of T |
| ΔT_ref | T0 | characteristic scale |
| t_eff | T0 or T3 | effective thickness for volumetric charge |

### 5. Implementation Constraints

- C_def **remains** dimensionless (tier TX).
- δρ_eff must use mean subtraction (not raw ρ_eff).
- All new parameters classified in the parameter registry.
- No output presented as calibrated physical prediction.

## Consequences

### Positive
- Advances the MVP toward the process-to-device bridge.
- Provides a testable electrostatic disorder model.
- Maintains the perturbation philosophy (Boussinesq-inspired).
- Manufactured solutions enable rigorous numerical verification.

### Negative
- Increases risk of physical overclaim.
- Introduces uncalibrated parameters (D_it_ref, λT, λC).
- Requires careful documentation of limitations.
- May be misinterpreted by external readers of the public repository.

### Mitigations
- Parameter registry with tier system (T0/T3/TX).
- Quality gates blocking forbidden scope terms.
- Explicit non-goals in this ADR.
- README and docs disclaim physical accuracy.

---

## Explicit Non-Goals

The following are **explicitly out of scope** for v0.4:

| Non-Goal | Rationale |
|----------|-----------|
| Schrödinger equation | Quantum confinement not modeled |
| TCAD/QTCAD integration | External tool dependency |
| Coherence time prediction | Requires spin Hamiltonian |
| Charge noise spectral density | Requires fluctuator models |
| Gate fidelity prediction | Requires quantum control theory |
| Experimental calibration | No experimental data available |
| Real device geometry | 2D demonstrative domain only |
| Real chip performance claims | Explicitly forbidden |
| Self-consistent Fermi level | Requires iterative Poisson-Schrödinger |
| Energy-resolved trap occupancy | Deferred to v0.5+ |
