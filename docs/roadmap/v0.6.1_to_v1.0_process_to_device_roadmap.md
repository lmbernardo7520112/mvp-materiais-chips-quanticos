## v0.6.2 — Curated E1/E2 D_it(E) Profile Library

**Type:** Documentation + code.
**Scope:** Literature-informed D_it(E) profiles for common Si/SiO₂ interfaces.

**Deliverables:**
- E1 nominal Si/SiO₂ thermal oxide profile (literature-informed).
- E1 high-trap stress profile (radiation or hot-carrier analogy).
- E2 metadata schema validation.
- Profile factory/registry pattern.
- No new physics coupling.

**References:**
- Fleetwood (2018), IEEE TNS: TID-induced D_it.
- Lenahan & Conley (1998): Pb centers.
- Cartier et al. (1993): hot-carrier D_it.

## v0.7.0 — ADR-C2: Surface-to-Volume Charge Mapping

**Type:** Documentation-only (ADR proposal).
**Scope:** Define the mapping σ_eff → ρ_eff.

**Key questions to resolve:**
1. What is t_eff? Physical oxide thickness or numerical smearing?
2. What geometry? 1D slab, 2D cross-section?
3. What boundary conditions for the charge layer?
4. What units? ρ_eff in C·m⁻³.
5. What remains non-calibrated?

## v0.7.1 — C2 RED/GREEN (if ADR accepted)

**Type:** Code (TDD).
**Pre-requisite:** ADR-C2 accepted.
**Scope:** Implement ρ_eff = σ_eff / t_eff with full TDD cycle.

## v0.8.0 — Poisson Coupling Demonstrator

**Type:** Code.
**Pre-requisite:** C2 implemented and validated.
**Scope:** Couple ρ_eff to the existing Poisson solver.

**Deliverables:**
- Feed ρ_eff from C1→C2 chain into ∇·(ε∇φ) = −ρ_eff.
- Demonstrate φ(x,y) from defect-derived charge.
- No physical interpretation of φ without explicit disclaimers.

## v0.9.0 — Schrödinger Effective Confinement Prototype

**Type:** Code.
**Pre-requisite:** Poisson coupling demonstrator validated.
**Scope:** 1D/2D Schrödinger solver for effective confinement.

**Deliverables:**
- V_conf(x) = −q_e × φ(x) or equivalent.
- Solve (−ℏ²/2m_eff)∇²ψ + V_conf·ψ = Eψ.
- Extract ground state and first excited state.
- No coherence claims.

## v1.0 — Integrated Process-to-Device Demonstrator

**Type:** Integration.
**Pre-requisite:** All above validated.
**Scope:** End-to-end pipeline demonstration.

**Deliverables:**
- Complete chain: process → defects → charge → potential → confinement.
- Audit trail for each layer.
- Final report with explicit limitations.
- No calibration claims.

## Stop Conditions

Progress to the next layer is **blocked** if any of the following
conditions exist:

1. No accepted ADR for the target layer.
2. Missing TDD RED tests before implementation.
3. Calibration claims in any artifact.
4. Missing dimensional/unit audit.
5. Solver coupling without C2 acceptance.
6. Schrödinger without Poisson coupling validation.
7. Quality gates failing.
8. Coverage below 70%.
