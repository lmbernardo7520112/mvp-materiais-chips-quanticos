## Gate Resolution Protocol

1. **MUST gates** devem todos estar verdes antes de qualquer push.
2. **SHOULD gates** devem estar verdes se o feature correspondente foi implementado.
3. Gates falhantes de MUST → bloqueiam release.
4. Gates falhantes de SHOULD → documentar em technical_debt.md e deferir.
5. **Nenhum gate pode ser desabilitado** sem ADR.
