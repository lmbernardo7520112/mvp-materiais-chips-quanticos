# v0.4.6 Runtime Scale Metadata — TDD Plan

> **Date:** 2026-05-13
> **Status:** RED
> **ADR Reference:** ADR-008 (Accepted in v0.4.3)
> **Preceding Release:** v0.4.5 (documentation-only review)
> **Council Decision:** 5/5 unanimous Option B for future implementation

---

## Goal

Implement, in a future GREEN phase, metadata-only runtime declaration for the
existing bridge outputs.

This means the Poisson Bridge CSV (`poisson_bridge_metrics.csv`) will include
safe metadata columns declaring the current scale regime, without changing any
numerical values or solver behavior.

## Non-goals

- No numerical physics change.
- No solver change.
- No epsilon substitution (epsilon remains 1.0 in the bridge).
- No D_it_SI.
- No sigma_eff.
- No rho_eff physical conversion.
- No t_eff physical conversion.
- No delta_E_window.
- No physical phi interpretation.
- No Option C.
- No Schrödinger equation.
- No Poisson non-linear.
- No self-consistency loop.
- No Majorana physics.

## Intended future behavior

Future implementation should add safe metadata to runtime artifacts:

- `scale_mode` = `demonstrative`
- `geometry_mode` = `normalized_2d`
- `source_mode` = `demonstrative`
- `physical_interpretation_allowed` = `false`
- `literature_scaled_constants_available` = `true`
- `option_c_enabled` = `false`
- `numerical_values_modified` = `false`

## Required invariants

1. Existing numeric metrics remain unchanged (max_abs_delta_rho_eff, mean_delta_rho_eff, max_abs_phi, solver_iterations, solver_residual, converged).
2. Existing figures remain generated (poisson_bridge_potential.png).
3. Existing CSV remains readable by any consumer expecting the original 6 columns.
4. Metadata fields must not imply calibration.
5. `physical_interpretation_allowed` must remain `False`.
6. `option_c_enabled` must remain `False`.
7. No physical unit interpretation of φ is authorized.

## Future implementation sketch

Allowed in future GREEN phases:

- Add `scale_metadata_to_record()` serialization helper to `scale_modes.py`.
- Add `attach_scale_metadata_to_metrics()` helper to `scale_modes.py`.
- Add metadata-only fields to `run_poisson_bridge.py` output CSV.
- Add non-regression tests proving numeric columns unchanged.

Not allowed:

- Changing numerical solver.
- Changing `effective_charge.py`.
- Changing epsilon.
- Implementing physical charge mapping.
- Implementing any Option C parameter.
