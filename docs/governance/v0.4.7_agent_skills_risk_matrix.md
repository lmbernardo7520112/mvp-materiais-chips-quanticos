## R1 — Skill Crystallizes Wrong Scientific Rule

- **Severity:** Critical
- **Likelihood:** Low
- **Description:** A skill might encode an incorrect scientific rule
  (e.g., wrong dimensional chain, incorrect unit conversion factor),
  which the agent would then follow consistently — amplifying the error.
- **Mitigation:** Skills are Markdown-only playbooks with no executable
  code. All dimensional examples use established CODATA/NIST values.
  Physics-dimensional-audit skill includes explicit verification steps.
  Any future executable scripts require dedicated PR review.
- **Related Gate:** G4, G9
- **Owner:** Project maintainer

---

## R2 — Skill Causes Agent to Skip Human/Council Review

- **Severity:** High
- **Likelihood:** Medium
- **Description:** A skill might instruct the agent to proceed with
  implementation without first obtaining human approval or council
  deliberation for scope-changing decisions.
- **Mitigation:** scope-guardrails skill explicitly requires ADR/council
  for any new physics. tdd-red-green-release skill requires user approval
  before merge. No skill authorizes skipping human review.
- **Related Gate:** G4
- **Owner:** Project maintainer

---

## R3 — Skill Hides Scope Creep Behind Automation

- **Severity:** High
- **Likelihood:** Medium
- **Description:** Automated governance might create a false sense of
  security, allowing scope creep to go unnoticed because "the skill
  said it was okay."
- **Mitigation:** report-auditor skill explicitly checks for scope creep.
  ai-rse-gateops skill requires `git diff --name-status` evidence.
  Gate G7 verifies no unauthorized file changes.
- **Related Gate:** G7, G8
- **Owner:** Project maintainer

---

## R4 — External Skill Introduces Unsafe Workflow

- **Severity:** Critical
- **Likelihood:** Low (mitigated by G6)
- **Description:** An external skill package might introduce workflows
  that bypass project-specific guardrails or introduce dependencies.
- **Mitigation:** Gate G6 prohibits external skill installation. This
  release uses only project-scoped, version-controlled skills. No
  `pyproject.toml` changes.
- **Related Gate:** G6
- **Owner:** Project maintainer

---

## R5 — Executable Skill Script Performs Destructive Operation

- **Severity:** Critical
- **Likelihood:** Low (mitigated by G5)
- **Description:** An executable script inside a skill directory might
  delete files, modify policy, or perform other destructive operations
  without review.
- **Mitigation:** Gate G5 prohibits executable scripts in this release.
  Only `SKILL.md` files are allowed. Future scripts require dedicated
  PR with acceptance gates.
- **Related Gate:** G5
- **Owner:** Project maintainer

---

## R6 — Skill Conflicts with Quality Gates

- **Severity:** Medium/High
- **Likelihood:** Low
- **Description:** A skill instruction might contradict the quality gates
  configuration in `policy.json`, leading to inconsistent behavior.
- **Mitigation:** Skills reference `policy.json` as the source of truth
  for forbidden terms, authorized files, and stage configuration. Gate
  G10 verifies quality gates pass. Skills do not override policy.
- **Related Gate:** G10
- **Owner:** Project maintainer

---

## R7 — Skill Uses Stale Roadmap Assumptions

- **Severity:** High
- **Likelihood:** Medium
- **Description:** Skills encode current project state (v0.4.x, Option B,
  demonstrative mode). If the project advances to v0.5 or Option C,
  skills must be updated — but might be forgotten.
- **Mitigation:** Skills reference versioned ADRs and policy.json rather
  than hardcoding roadmap state. scope-guardrails skill lists deferred
  items with "requires ADR" markers. Skills are version-controlled and
  reviewed in PRs.
- **Related Gate:** G4
- **Owner:** Project maintainer

---

## R8 — Skill Encourages Implementation Without ADR

- **Severity:** Critical
- **Likelihood:** Low
- **Description:** A skill might inadvertently encourage the agent to
  implement a feature that requires an ADR but doesn't have one yet.
- **Mitigation:** scope-guardrails skill explicitly lists all items
  requiring ADR authorization. The decision flow diagram in the skill
  requires stopping and creating governance documents before code.
- **Related Gate:** G9
- **Owner:** Project maintainer

---

## R9 — Skill Bloats Context or Reduces Reasoning Quality

- **Severity:** Medium
- **Likelihood:** Medium
- **Description:** Too many or too verbose skills might overwhelm the
  agent's context window, reducing reasoning quality on the actual task.
- **Mitigation:** Skills are structured as checklists with clear headings.
  Each skill has a "When to Use" section to avoid unnecessary activation.
  Six skills is a manageable count for the current project scope.
- **Related Gate:** G4
- **Owner:** Project maintainer

---

## R10 — Skill Duplicates policy.json Inconsistently

- **Severity:** Medium/High
- **Likelihood:** Low
- **Description:** Skills that reference forbidden terms or authorized
  files might duplicate `policy.json` content and drift out of sync.
- **Mitigation:** Skills reference `policy.json` as the authoritative
  source rather than duplicating its content. scope-guardrails skill
  describes categories (e.g., "terms prohibited by policy.json") rather
  than listing specific terms. Gate G8 verifies policy is unchanged.
- **Related Gate:** G8
- **Owner:** Project maintainer
