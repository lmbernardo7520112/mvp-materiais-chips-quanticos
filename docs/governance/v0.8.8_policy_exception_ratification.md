# v0.8.8 Policy Exception Ratification

## Decision

Human ratifies a minimal governance exception to update `tools/quality_gates/policy.json`.

## Reason

The RED test file authorized by ADR-016 must reference the term "Poisson". The quality gate treats "Poisson" as conditionally forbidden unless the file is listed in `authorized_files`.

## Scope of Exception

Allowed:

* Add `test_one_way_poisson_coupling_red.py` to `authorized_files` only.

Not allowed:

* Remove forbidden terms.
* Remove conditionally forbidden terms.
* Weaken gates.
* Authorize src changes.
* Authorize scripts changes.
* Authorize runtime.
* Authorize `solve_poisson`.
* Authorize physical phi.
* Authorize implementation.
* Authorize paid API.
* Authorize SDK.
* Authorize goal-like autonomous execution.

## Audit Evidence

The `git diff main -- tools/quality_gates/policy.json` confirms:

* Only the `authorized_files` array in the `v0.5` stage was modified.
* One line added: `"test_one_way_poisson_coupling_red.py"`.
* No other fields changed.

## Classification

This is a governance authorization update, not a scientific implementation.

## Expiration / Future Review

The exception remains valid only for the RED file and must be revisited before GREEN if the file is renamed, moved, or if production runtime is introduced.
