# v0.7.0 C2 Regularization Evidence Taxonomy

## Purpose

Define evidence levels for future interface-source and regularization
choices in the C2 layer (σ_eff → electrostatic source term).

This taxonomy extends the project's evidence policy (ADR-010, v0.5.6
amendment) to cover regularization-specific decisions.

## Levels

| Level | Name | Meaning | Allowed now? |
|-------|------|---------|:------------:|
| S0 | Synthetic conservation fixture | Test-only conservation fixture for verifying ∫ρ dV = ∫σ dA | Future tests only |
| R1 | Numerical regularization length | l_reg as numerical smearing parameter, not physical thickness | Future fallback |
| E1 | Literature-informed depth envelope | Literature-based depth/distribution for sensitivity studies | Conditional |
| E2 | Experimental depth/distribution prior | Specific source with complete metadata (technique, uncertainty) | Conditional |
| E3 | Same-stack physical depth model | Depth model from comparable stack/process conditions | Blocked |
| E4 | Calibrated device-fitted depth | Depth fitted to specific target device measurements | Blocked |

## Rules

### S0 — Synthetic conservation fixture

- Used only in test suites to verify charge conservation.
- Must not appear in operational code.
- Example: `test_integral_rho_matches_integral_sigma`.

### R1 — Numerical regularization length

- l_reg is a numerical parameter, not a physical thickness.
- Must be reported in output metadata.
- Results must be tested for sensitivity to l_reg.
- Must satisfy conservation: ρ_reg × l_reg = σ_eff.
- Must not be called "t_eff" or "physical thickness."

### E1 — Literature-informed depth envelope

- Provides scale envelope for sensitivity studies.
- Does not calibrate the MVP.
- Does not override C2-A sheet-source primary path.
- Must declare source, trap_family, material_stack.
- Must declare calibration_status = "not_calibrated".

### E2 — Experimental depth/distribution prior

- Provides specific depth distribution from measurement.
- Requires complete metadata (see ADR-012 C2-C metadata table).
- Does not calibrate the MVP.
- Must declare calibration_status = "not_calibrated".

### E3/E4 — Blocked

- Require same-stack or device-specific experimental data.
- Beyond current MVP scope.
- Require their own future ADR.

## Prohibited Patterns

- Treating R1 (l_reg) as physical t_eff.
- Using E1/E2 depth priors to claim calibration.
- Implementing E3/E4 without dedicated ADR.
- Converting the MVP into a calibrated device simulator.
- Confusing interface-state D_it profiles with border/oxide trap
  spatial distributions without declaring trap_family.
