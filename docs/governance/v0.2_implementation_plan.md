### 0.4 Decisões da ADR-005 que afetam v0.2

A ADR-005 define v0.2 como **robustez numérica**, não física nova:

- 2D térmico simplificado.
- Avaliação rígida de estabilidade 2D.
- Análise de convergência dx/dt.
- pytest-cov.
- **Zero física nova pesada.**

### 0.5 Por que v0.2 deve focar em robustez numérica

1. O pipeline process-to-device (v0.3–v0.5) exigirá solvers 2D acoplados.
2. Sem Domain2D, solver 2D e convergência validada, as ondas v0.3+ não terão
   infraestrutura estável para novas equações (Poisson, Schrödinger).
3. pytest-cov é necessário para medir cobertura real antes de adicionar complexidade.
4. Convergência dx/dt é obrigatória para demonstrar confiabilidade numérica.

### 0.6 Por que v0.2 NÃO deve implementar física nova pesada

- **Poisson 2D** requer definição de ρ_eff, que depende de C_def (v0.3).
- **Schrödinger** depende do potencial eletrostático resolvido (v0.4).
- **TCAD/QTCAD** quebra a agilidade NumPy (ADR-001) e exige validação externa.
- **Czochralski real** é tese de doutorado; fora do escopo do MVP.
- **Defect-like reaction-diffusion** requer termos G(T) e R(T,C) que são escopo v0.3.
- Implementar qualquer desses sem base 2D validada criaria debt técnico destrutivo.

## 9. Decisões Técnicas

### 9.1 Decisão sobre 2D

O "2D simplificado" significa:
- Grade retangular uniforme (nx × ny).
- Euler explícito (sem ADI nesta versão).
- BCs Dirichlet para térmico; Neumann no-flux para difusivo.
- Sem geometrias complexas, curvilíneas ou não-estruturadas.
- O domínio é demonstrativo — não representa uma fatia real de wafer.

### 9.2 Decisão sobre análise de convergência

**Caso de referência analítico — equação do calor 2D com Dirichlet homogêneo:**

```
T(x,y,t) = sin(π·x/Lx) · sin(π·y/Ly) · exp(-α·π²·(1/Lx² + 1/Ly²)·t)
```

- BCs: T = 0 em todas as fronteiras (Dirichlet homogêneo).
- CI: T₀(x,y) = sin(π·x/Lx) · sin(π·y/Ly).
- A solução decai exponencialmente para zero.
- Erro medido contra a solução analítica em tempo fixo t_final.

**Saídas obrigatórias:**
- `convergence_results.csv` com colunas: `nx, ny, dx, dy, dt, error_l2, error_linf, observed_order, elapsed_time`.
- `convergence_analysis.png` (log-log error vs dx).

**Critério:**
- `observed_order >= 1.5` em pelo menos uma faixa de refinamento válida.
- Se fronteiras ou discretização temporal impedirem a taxa O(dx²) esperada,
  registrar como **limitação técnica documentada**, não mascarar.

**Sequência de refinamento sugerida:** nx = ny ∈ {11, 21, 41, 81}.

### 9.3 Decisão sobre pytest-cov

- Adicionar `pytest-cov>=4.0,<6.0` ao `[project.optional-dependencies] dev`.
- Meta mínima: **≥70%** de cobertura de linhas em `src/mvp_quantum_materials/`.
- **CI DEVE falhar** se coverage < 70%.
- Comando planejado:
  ```bash
  pytest --cov=mvp_quantum_materials --cov-report=term-missing --cov-fail-under=70
  ```
- Coverage é condição necessária, não suficiente. Testes devem verificar
  propriedades físicas/numéricas (campo constante, suavização, finitude,
  rejeição de instabilidade), não apenas execução.

### 9.4 Decisão sobre notebooks / Jupytext

- Classificado como SHOULD.
- Se implementado: usar Jupytext (`.py:percent` format) para manter em controle de versão.
- Notebooks NÃO devem conter lógica principal — apenas chamam módulos de `src/`.
- Se não implementado, documentar em TD e deferir para v0.3.

### 9.5 Decisão sobre sensibilidade global

- Classificado como COULD.
- Se implementado: Morris screening como passo intermediário (mais simples que Sobol completo).
- Se não implementado: manter OAT e deferir Sobol/Morris completo para v0.3/v0.4.

### 9.6 Critério final de convergência

O solver 2D é aceito se:
1. Teste de campo constante demonstra erro de máquina (≤1e-12).
2. Convergência mostra taxa ≥ O(dx^1.5) (relaxado de O(dx²) para acomodar efeitos de fronteira).
3. Nenhum campo diverge (NaN/inf) nos cenários padrão.
