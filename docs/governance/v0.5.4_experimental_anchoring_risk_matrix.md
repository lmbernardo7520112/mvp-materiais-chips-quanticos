# v0.5.4 Experimental Anchoring Risk Matrix

---

## R1 — Literature Range Treated as Calibration

- **Severity:** Critical
- **Failure mode:** E1 literature values presented as calibrated model parameters.
- **Mitigation:** All outputs must declare `calibration_status = not_calibrated`.
  Evidence taxonomy labels must appear in CSV metadata.
- **Responsible skill:** report-auditor, scope-guardrails

## R2 — Experimental Curve from Incompatible Stack Transferred as Equivalent

- **Severity:** Critical
- **Failure mode:** D_it(E) from thermal SiO₂ on Si(100) applied to Si/SiGe
  heterostructure or different oxide without acknowledging mismatch.
- **Mitigation:** E2 metadata must include material stack, interface, and
  transferability limits. Source class must be labeled.
- **Responsible skill:** physics-dimensional-audit

## R3 — Energy Reference Mismatch

- **Severity:** Critical
- **Failure mode:** D_it(E) plotted vs. E−E_v in source but bins defined vs.
  E−E_c in model, causing systematic offset.
- **Mitigation:** Energy reference must be documented for every source.
  Conversion must be explicit and auditable.
- **Responsible skill:** physics-dimensional-audit

## R4 — eV/J Conversion Mismatch

- **Severity:** Critical
- **Failure mode:** Energy edges in eV mixed with density in J⁻¹·m⁻² without
  explicit conversion, producing ~10¹⁹ error in N_it.
- **Mitigation:** All internal computation in Joules. Conversion via
  `units.ELEMENTARY_CHARGE`. Tests must verify unit consistency.
- **Responsible skill:** physics-dimensional-audit

## R5 — Measurement Technique Sees Different Trap Population

- **Severity:** High
- **Failure mode:** DLTS deep-level data used for interface D_it(E) bins.
  DLTS probes bulk/deep traps, not interface states.
- **Mitigation:** Evidence review must classify technique and trap type.
  Only interface-state techniques qualify for D_it(E) bins.
- **Responsible skill:** physics-dimensional-audit

## R6 — Digitized Curve Overinterpreted

- **Severity:** High
- **Failure mode:** A published D_it(E) curve digitized to 100 points and
  treated as high-resolution profile, ignoring measurement uncertainty.
- **Mitigation:** Digitized curves must note resolution, uncertainty, and
  extraction model limitations.
- **Responsible skill:** report-auditor

## R7 — E2 Profile Used to Justify C2 Prematurely

- **Severity:** Critical
- **Failure mode:** An experimental-profile prior used to argue that the
  model is "experimentally informed enough" for ρ_eff/solver coupling.
- **Mitigation:** Acceptance gates explicitly block C2. E2 does not
  authorize ρ_eff, t_eff, or solver coupling.
- **Responsible skill:** scope-guardrails

## R8 — Missing Uncertainty Metadata

- **Severity:** High
- **Failure mode:** E2 source lacks error bars or confidence bounds,
  making it impossible to assess reliability.
- **Mitigation:** E2 classification requires uncertainty documentation.
  Sources without uncertainty are downgraded to E1.
- **Responsible skill:** report-auditor

## R9 — Smooth Profile Imposed Without Evidence

- **Severity:** Medium–High
- **Failure mode:** A Gaussian or polynomial fit applied to bins without
  justification, creating false smoothness.
- **Mitigation:** P2/P3 profiles remain deferred per ADR-010. Only P1
  piecewise-constant is recommended.
- **Responsible skill:** scope-guardrails

## R10 — Experimental Anchoring Confused with Device Prediction

- **Severity:** Critical
- **Failure mode:** σ_eff computed from literature-informed bins presented
  as a prediction of specific device charge.
- **Mitigation:** All outputs must declare `device_prediction_claimed = False`.
  Non-calibration statement must appear in all documentation.
- **Responsible skill:** report-auditor, scope-guardrails
