# v0.5.4 Energy Profile Evidence Taxonomy

## Purpose

Define evidence levels for candidate D_it(E) profiles to prevent false
calibration and ensure transparent provenance of bin values in future
P1 piecewise implementations.

## Evidence Levels

| Level | Name | Meaning | Operational? | Can calibrate? |
|-------|------|---------|-------------|----------------|
| S0 | TEST_ONLY | Synthetic fixture for mathematical verification | ❌ Tests only | No |
| E0 | Demonstrative | ~~Chosen for mathematical sanity only~~ | ❌ **Deprecated (v0.5.6)** | No |
| E1 | Literature-informed | Based on ranges/shapes reported in peer-reviewed literature | ✅ Minimum operational | No |
| E2 | Experimental-profile prior | Digitized or tabulated D_it(E) from a specific experiment/publication | ✅ Conditional | No |
| E3 | Same-stack calibration | Data from same or directly comparable material stack/process | ❌ Blocked | Not yet |
| E4 | Device-fitted inverse model | Fitted from target device response, noise, or transport data | ❌ Blocked | Future only |

## Rules (Updated v0.5.6)

1. **S0 TEST_ONLY** fixtures may be used only inside test suites for
   mathematical verification. They must not appear in release artifacts,
   benchmark CSVs, or model outputs.
2. **E0 demonstrative** is deprecated (v0.5.6 amendment). It is not accepted
   as an operational evidence level for model-facing D_it(E) profiles.
3. **E1 literature-informed** is the minimum operational evidence level for
   future P1 piecewise implementation.
4. **E2** requires full source metadata (see below) and uncertainty notes.
   Must be labeled as "experimental-profile prior" in all outputs. Does NOT
   authorize calibration claims.
5. **E3 and E4** require a dedicated ADR (beyond ADR-010) and are NOT
   authorized in the current roadmap.
6. **No evidence level** authorizes ρ_eff, t_eff, solver coupling,
   physical φ interpretation, or C2/C3 features.

## Required Metadata for E2+

Any profile classified E2 or higher must include:

| Field | Description |
|-------|-------------|
| Source | Full bibliographic reference |
| Material stack | e.g., Si/SiO₂, Si/SiGe, SOI |
| Interface | Which interface D_it was measured at |
| Technique | Measurement method (C-V, conductance, DLTS, etc.) |
| Temperature | Measurement temperature |
| Energy reference | Band edge, midgap, Fermi level, etc. |
| Energy range | Range of E over which D_it(E) was extracted |
| D_it units | Original units (eV⁻¹·cm⁻², J⁻¹·m⁻², etc.) |
| Uncertainty | If available, error bars or confidence bounds |
| Extraction assumptions | Model used to extract D_it from raw data |
| Transferability limits | Known limitations when applying to different stacks |
