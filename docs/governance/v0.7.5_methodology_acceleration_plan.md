# v0.7.5 Methodology Acceleration Plan

## Goal

Add project-scoped skills and templates to accelerate AI-RSE GateOps without reducing rigor.

## Added Skills

1. `release-acceleration-lanes` — Classify work into fast-lane or slow-lane release paths.
2. `minimal-red-contract` — Design minimal sufficient RED tests.
3. `dependency-governance` — Govern new dependencies with decision briefs.
4. `evidence-metrics-refresh` — Prevent stale or invented metrics.
5. `ai-failure-mode-review` — Review AI failure modes before commits.
6. `human-decision-log` — Capture critical human decisions.

## Added Templates

1. `docs/templates/adr_template.md`
2. `docs/templates/acceptance_review_template.md`
3. `docs/templates/tdd_red_plan_template.md`
4. `docs/templates/green_report_template.md`
5. `docs/templates/dependency_decision_template.md`
6. `docs/templates/release_closure_template.md`
7. `docs/templates/human_decision_log_entry_template.md`

## Added Governance Docs

- `docs/governance/release_methodology_checklist.md`
- `docs/governance/human_decision_log.md`

## Not Changed

- no src;
- no tests;
- no scripts;
- no policy;
- no pyproject;
- no physics;
- no solver coupling;
- no calibration claims.

## Expected Effect

- reduce repeated prompt size;
- reduce documentation overhead;
- preserve ADR/TDD/CI discipline;
- speed up v0.8.x C3 work.
