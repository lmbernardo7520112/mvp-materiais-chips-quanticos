## Risk Details

### R1 — Metadata Interpreted as Physical Calibration

- **Owner:** SMGS
- **Description:** A CSV containing `literature_scaled_constants_available=true` could be misread as "constants are applied to the simulation."
- **Mitigation:** Rename to `_module_exists`, add disclaimer in CSV header, enforce `physical_interpretation_allowed=False`.

### R2 — Runtime Metadata Perceived as Numerical Physics

- **Owner:** NMS
- **Description:** Adding metadata columns to CSV may be perceived as a numerical change, even if phi values are identical.
- **Mitigation:** Bitwise phi identity tests required before any metadata integration (future v0.4.6).

### R3 — Accidental Coupling of ε_r·ε₀ to Solver

- **Owner:** RSE
- **Description:** A developer replaces `epsilon=1.0` with `epsilon_r*epsilon_0` without TDD or ADR approval.
- **Mitigation:** Quality gate: `grep "from.*units.*import\|epsilon_0\|epsilon_r" src/mvp_quantum_materials/poisson_solver_2d.py` must return empty.

### R4 — Option C Introduced Through Metadata Pathway

- **Owner:** SMGS
- **Description:** Metadata integration provides a foothold for sneaking D_it_SI or σ_eff computation into the runtime.
- **Mitigation:** Prohibited terms quality gate. ADR amendment required before Option C.

### R5 — CSV Fields Misunderstood as Physical Outputs

- **Owner:** SDP
- **Description:** A field like `epsilon_r_Si=11.7` next to phi values suggests phi was computed with that permittivity.
- **Mitigation:** Only declare informational metadata, never values that could be confused with solver inputs.

### R6 — Inconsistent v0.4.4 Release Test Counts

- **Owner:** RSE
- **Description:** Release report v0.4.4 listed test_units.py=17, test_scale_modes.py=14. Actual counts: 12, 19.
- **Mitigation:** Correction document issued. Total 31 is correct; only per-file distribution was swapped.

### R7 — Gate Terminology Workaround

- **Owner:** SMGS
- **Description:** Defining gates in terms that technically pass but allow scope creep (e.g., "metadata" that includes numerical constants).
- **Mitigation:** Council review of gate definitions. Explicit prohibition list in acceptance gates.

### R8 — Future PR Skips TDD

- **Owner:** RSE
- **Description:** A future PR directly wires units.py to the solver without RED→GREEN TDD cycle.
- **Mitigation:** Branch protection requires CI. Quality gate grep for solver imports from units.py. Council must re-approve.
