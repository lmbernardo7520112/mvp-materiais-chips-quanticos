# v0.5.3 C1 Energy-Distribution Risk Matrix

---

## R1 — D_it(E) Treated as Calibrated Distribution

- **Severity:** High
- **Description:** Users or reviewers may interpret a piecewise D_it(E)
  profile as a calibrated measurement of interface trap distribution.
- **Mitigation:** All metadata must declare `calibration_status = "not_calibrated"`.
  Documentation must state that D_it(E) bins are literature-anchored
  demonstrations, not experimental fits.
- **Responsible skill:** report-auditor, scope-guardrails

## R2 — Gaussian Profile Creates False Physical Specificity

- **Severity:** High
- **Description:** A Gaussian D_it(E) with specific A, E₀, σ_E may appear
  to encode knowledge about the true trap distribution.
- **Mitigation:** Gaussian (P2) is deferred until experimental justification
  exists. Only P1 piecewise-constant is recommended for initial implementation.
- **Responsible skill:** physics-dimensional-audit, scope-guardrails

## R3 — Energy Axis Confusion Between eV and J

- **Severity:** Critical
- **Description:** Mixed use of eV and J in energy edges could produce
  errors of ~10¹⁹ in N_it.
- **Mitigation:** All internal computation must use Joules. Conversion
  from eV must use `units.ELEMENTARY_CHARGE` explicitly. Tests must verify
  unit consistency.
- **Responsible skill:** physics-dimensional-audit

## R4 — Piecewise Bins Overlap or Leave Gaps Silently

- **Severity:** Medium–High
- **Description:** If energy edges are not properly ordered or contain
  duplicates, the integration produces incorrect N_it.
- **Mitigation:** Implementation must validate that edges are strictly
  increasing. Tests must reject unordered or repeated edges.
- **Responsible skill:** tdd-red-green-release

## R5 — Negative D_it Values Accidentally Allowed

- **Severity:** High
- **Description:** Negative D_it has no physical meaning and would produce
  incorrect N_it (potentially negative), undermining dimensional sanity.
- **Mitigation:** Implementation must validate D_i ≥ 0 for all bins.
  Tests must reject negative values.
- **Responsible skill:** physics-dimensional-audit

## R6 — Integrated N_it Compared as if Experimental

- **Severity:** High
- **Description:** Computed N_it from piecewise integration may be compared
  against experimental trap counts as if it were a prediction.
- **Mitigation:** Non-calibration metadata must be enforced. Documentation
  must state this is dimensional bookkeeping, not measurement prediction.
- **Responsible skill:** report-auditor

## R7 — Energy-Distribution Upgrade Used to Justify C2 Prematurely

- **Severity:** Critical
- **Description:** Adding D_it(E) could be interpreted as sufficient
  physics to justify ρ_eff, t_eff, or solver coupling.
- **Mitigation:** ADR-010 explicitly does NOT authorize C2 features.
  Acceptance gates G9–G11 enforce ρ_eff/t_eff/solver blocking. The
  C1/C2 boundary is that σ_eff remains the terminal output.
- **Responsible skill:** scope-guardrails

## R8 — Literature Profile Overfitted Without Data

- **Severity:** High
- **Description:** Piecewise D_it(E) bin values could be tuned to match
  a desired σ_eff, creating a false sense of physical fidelity.
- **Mitigation:** All bin values must use literature-anchored ranges
  (T2/T3 evidence tiers). The CSV must declare `not_calibrated`.
  Benchmark scenarios should use the same D_it ranges as v0.5.2.
- **Responsible skill:** report-auditor, physics-dimensional-audit
