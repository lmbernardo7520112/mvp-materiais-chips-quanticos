# v0.8.0 C3 Solver Coupling Risk Matrix

| Risk ID | Risk Description | Severity | Failure Mode | Mitigation | Responsible Skill | Release Gate |
|---|---|---|---|---|---|---|
| R1 | Solver coupling before projection contract. | Critical | Monolithic coupling masks numerical bugs in the source mapping, leading to false physics claims. | Adopt C3-A as intermediate grid projection step. No solver until this is RED/GREEN validated. | ai-rse-gateops | G15 |
| R2 | Charge not conserved during projection. | Critical | Sum of projected charges differs from C2 source total charge. Erroneous physical effect results. | Mandate projection_preserves_total_charge RED tests for C3 implementation. | physics-dimensional-audit | G24 |
| R3 | physical phi inferred prematurely. | Critical | Using a non-convergent, non-self-consistent Poisson run to claim physical electrostatic potential. | Block physical phi via `physical_interpretation_allowed=False`. | scope-guardrails | G17 |
| R4 | boundary metadata absent. | Critical | Solver assumes default boundaries (e.g. Dirichlet vs Neumann) implicitly, altering output. | `boundary_condition_metadata` is mandatory parameter for C3 projection. | physics-dimensional-audit | G24 |
| R5 | grid resolution artifacts. | High | A coarse grid aliases the C2 interface sheet, causing charge lumps. | Introduce `grid_spacing` checks and resolution dependency warnings. | minimal-red-contract | G24 |
| R6 | interface sheet smeared incorrectly. | High | The volumetric regularization projects as a smeared sheet, reducing local field accuracy near the interface. | C3-A explicitly handles smeared projection; plan C3-B as a future jump-condition correction. | physics-dimensional-audit | G11, G12 |
| R7 | C2 metadata lost. | High | Provenance of the charge (e.g. from D_it(E)) is severed. | C3 RED test `test_projection_preserves_c2_metadata` ensures metadata propagation. | ai-rse-gateops | G24 |
| R8 | l_reg sensitivity hidden. | High | Regularization length dependency is not passed forward to the solver. | `l_reg_sensitivity` must be logged in C3 metadata. | evidence-metrics-refresh | G24 |
| R9 | one-way Poisson run treated as validation. | Critical | Developers take the result of C3-C as a fully coupled model. | Explicit labeling as 'one-way demonstrator' and blocking self-consistent loop (C3-D). | scope-guardrails | G13, G14 |
| R10 | AI-for-Science track confused with classical solver. | High | ML experiments accidentally affect the projected source. | Complete namespace separation per ADR-014. | scope-guardrails | G19 |
| R11 | solver existing assumptions conflict with projected source. | High | Poisson solver expecting volume sources receives interface conditions incorrectly. | Step C3-A converts interface to volume discretely before Poisson coupling. | ai-rse-gateops | G11 |
| R12 | future quantum confinement inherits projection artifacts. | High | Schrödinger equation solved on an erroneous Poisson potential. | C3 projection must be proven conserved and robust before future coupling (C3-C). | scope-guardrails | G18 |
