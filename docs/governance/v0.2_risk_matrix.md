## R-01: Instabilidade numérica 2D

- **Probabilidade:** Média
- **Impacto:** Médio
- **Descrição:** Euler explícito 2D tem critério de estabilidade mais restritivo
  que 1D. dt_max diminui com o quadrado da resolução. Malhas finas podem
  exigir milhões de passos temporais.
- **Mitigação:** Implementar guard de estabilidade 2D que recusa dt instável
  com ValueError. Teste T-2D-05 obrigatório. Malhas demonstrativas com nx, ny ≤ 101.
- **Gate associado:** G-08 (solver 2D recusa dt instável)

---

## R-02: Aumento excessivo de escopo

- **Probabilidade:** Alta
- **Impacto:** Alto
- **Descrição:** Tentação de implementar difusão 2D, notebooks, sensibilidade global,
  convergência e Poisson tudo de uma vez.
- **Mitigação:** Classificação MUST/SHOULD/COULD/WON'T rigorosa. SHOULD e COULD
  só são implementados após MUST estar verde. Poisson é WON'T explícito.
- **Gate associado:** G-19 (nenhuma física nova pesada)

---

## R-03: Quebra de regressão v0.1

- **Probabilidade:** Alta
- **Impacto:** Alto (mas mitigável)
- **Descrição:** Novos módulos podem interferir com imports, configs ou fixtures existentes.
- **Mitigação:** Solvers 2D em módulos **separados** (thermal_solver_2d.py).
  Nenhuma modificação em thermal_solver.py ou diffusion_solver.py existentes.
  Gate NR-01 obrigatório: rodar testes v0.1 isoladamente.
- **Gate associado:** G-05 (todos os testes v0.1 passam)

---

## R-04: Interpretação indevida de C como físico

- **Probabilidade:** Média
- **Impacto:** Alto
- **Descrição:** Figuras 2D podem dar aparência de simulação real. Usuários
  externos podem interpretar C como concentração física calibrada.
- **Mitigação:** Disclaimers obrigatórios em títulos de figuras, docstrings, README.
  L-01 permanece. C = proxy adimensional em toda documentação v0.2.
- **Gate associado:** G-20 (2D não apresentado como wafer real)

---

## R-05: Figuras 2D enganosas

- **Probabilidade:** Baixa
- **Impacto:** Médio
- **Descrição:** Contour plots 2D de T(x,y) ou C(x,y) podem parecer
  visualmente convincentes para audiência não-técnica.
- **Mitigação:** Todos os títulos de figuras 2D devem incluir
  "[demonstrativo — não calibrado]". Documentação interpreta cada figura
  com seção "O que NÃO pode ser inferido", como feito no relatorio_30_dias.md.
- **Gate associado:** G-20 (2D não apresentado como wafer real)

---

## R-06: pytest-cov como métrica cega

- **Probabilidade:** Média
- **Impacto:** Médio
- **Descrição:** Coverage ≥70% pode ser atingido com testes triviais que não
  verificam comportamento real. "100% coverage, 0% confidence."
- **Mitigação:** Coverage é condição necessária, não suficiente. Testes devem
  verificar propriedades físicas/numéricas (campo constante, suavização,
  finitude, rejeição de instabilidade), não apenas execução.
  Meta de 70% é piso, não teto.
- **Gate associado:** G-04 (coverage ≥70%)

---

## R-07: Sensibilidade global adicionando complexidade demais

- **Probabilidade:** Baixa
- **Impacto:** Baixo
- **Descrição:** Morris screening ou Sobol podem exigir centenas de simulações.
  Em 2D, cada simulação é mais cara.
- **Mitigação:** Classificado como COULD. Implementar somente após MUST e SHOULD
  verdes. Se tempo computacional for proibitivo, deferir para v0.3.
- **Gate associado:** Nenhum gate obrigatório — é COULD.

---

## R-08: Notebooks criando lógica paralela

- **Probabilidade:** Média
- **Impacto:** Baixo
- **Descrição:** Notebooks podem acumular lógica duplicada que diverge dos
  módulos src/. "Notebook rot."
- **Mitigação:** Notebooks são SHOULD. Se implementados, usam Jupytext (`.py:percent`)
  e chamam exclusivamente funções de `src/mvp_quantum_materials/`.
  Nenhuma lógica principal em notebooks. CI pode validar notebooks com `jupytext --test`.
- **Gate associado:** Nenhum gate obrigatório — é SHOULD.

## R-09: Tentação de antecipar v0.3/v0.4

- **Probabilidade:** Média
- **Impacto:** Alto
- **Descrição:** Durante implementação 2D, tentação de adicionar G(T), R(T,C),
  ρ_eff, Poisson ou outros termos da roadmap v0.3+.
- **Mitigação:** WON'T explícito. Toda física nova requer ADR.
  Revisão de PR deve verificar que nenhum solver novo além do planejado foi adicionado.
- **Gate associado:** G-19 (nenhuma física nova pesada)

---

## R-10: Falha de CI por aumento de tempo computacional

- **Probabilidade:** Baixa
- **Impacto:** Médio
- **Descrição:** Testes 2D podem ser significativamente mais lentos que 1D.
  Se o CI timeout for atingido, pipeline quebra.
- **Mitigação:** Usar malhas pequenas nos testes (nx, ny ≤ 21 para testes rápidos).
  Malhas maiores apenas em scripts de geração de resultados, não em pytest.
  Monitorar tempo de CI e ajustar se necessário.
- **Gate associado:** G-17 (CI verde)

---

## R-11: Violação da política additive-only em APIs 1D

- **Probabilidade:** Média
- **Impacto:** Alto
- **Descrição:** Extensão de `domain.py` ou `config.py` pode inadvertidamente
  alterar assinaturas existentes de Domain1D ou ThermalConfig, quebrando
  testes v0.1 e artefatos.
- **Mitigação:** Gate G-26 obrigatório: `git diff main` nos arquivos 1D deve
  ser vazio para solvers e additive-only para domain/config.
  Revisão de PR deve verificar que nenhuma assinatura existente mudou.
  Testes v0.1 rodam isoladamente antes de qualquer merge.
- **Gate associado:** G-26 (APIs 1D preservadas)
