# v0.5.3 Future TDD Plan — C1 Energy Distribution

**Status:** Plan only — no tests created in this release.

**Related ADR:** ADR-010 (Proposed)

---

## Purpose

Define the minimum RED test specifications for a future C1 energy-distribution
implementation. These tests are NOT created in v0.5.3. They will be created
in a future release once ADR-010 is accepted.

## Future RED Tests

### 1. test_constant_profile_matches_current_c1

Verify that P0 (constant profile) reproduces the existing C1 result:

```
N_it = D_it_SI × ΔE_window
```

A single-bin piecewise call with density=D_it_SI and edges=[E1, E2]
must produce identical N_it to `compute_nit_areal_density`.

### 2. test_piecewise_profile_integrates_segment_sum

Verify that a multi-bin piecewise profile produces:

```
N_it = Σᵢ D_i × (E_{i+1} - E_i)
```

Use 3–5 bins with known densities and verify exact sum.

### 3. test_piecewise_profile_requires_ordered_energy_edges

Energy edges must be strictly increasing: E₁ < E₂ < ... < E_n.

Must raise `ValueError` on unordered or repeated edges.

### 4. test_piecewise_profile_rejects_negative_density

All D_i values must satisfy D_i ≥ 0.

Must raise `ValueError` on negative density.

### 5. test_energy_units_are_joules

Energy edges must be in Joules (SI).

If input is in eV, conversion must be explicit via existing
`units.ELEMENTARY_CHARGE` constant.

### 6. test_integrated_nit_units_and_values

Output N_it must be in m⁻² and positive for D_i > 0 with finite window.

### 7. test_sigma_eff_from_integrated_nit

The downstream chain must be unchanged:

```
σ_eff = s_charge × q_e × N_it × f_occ
```

N_it from piecewise integration must plug into `compute_sigma_eff`
identically to N_it from constant approximation.

### 8. test_no_rho_eff_no_t_eff_no_solver_coupling

Implementation must NOT import or compute:
- ρ_eff
- t_eff
- Poisson solver calls
- Physical φ

### 9. test_gaussian_profile_deferred

Gaussian profile (P2) must NOT be implemented in the first P1 release.

If called, must raise `NotImplementedError` or similar.

### 10. test_non_calibration_metadata

All CSV outputs must continue to declare:
- `calibration_status = "not_calibrated"`
- `device_prediction_claimed = False`
- `solver_coupled = False`
- `rho_eff_present = False`
- `t_eff_present = False`
