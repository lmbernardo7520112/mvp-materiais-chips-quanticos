# v0.7.0 C2 Future RED Plan

## Trigger

This RED plan is activated **only** if ADR-012 is later accepted.

No tests or code may be created until ADR-012 status changes from
"Proposed" to "Accepted."

## Candidate Future Tests

These tests would be written in a future v0.7.2 RED phase:

### C2-A path (sheet source)

1. `test_sheet_source_representation_exists_only_after_acceptance`
   — Module exists only after ADR acceptance.

### C2-B path (volume regularization)

2. `test_conservative_regularization_preserves_total_charge`
   — ∫ρ dV = ∫σ dA within numerical tolerance.

3. `test_l_reg_required_for_volume_regularization`
   — Volume regularization must receive l_reg parameter.

4. `test_l_reg_positive_finite`
   — l_reg must be > 0 and finite.

5. `test_rho_reg_units_are_c_per_m3`
   — Output ρ_reg has units [C/m³].

6. `test_sigma_units_are_c_per_m2`
   — Input σ_eff has units [C/m²].

7. `test_integral_rho_matches_integral_sigma`
   — Numerical integral of ρ over volume matches σ × area.

8. `test_positive_negative_charge_sign_preserved`
   — Charge sign convention preserved through mapping.

### Scope guards

9. `test_no_solver_import_in_mapping_module`
   — C2 mapping module must not import solver functions.

10. `test_no_quantum_confinement_solver_import`
    — No quantum confinement solver imports.

### Evidence policy

11. `test_e1_depth_prior_requires_metadata`
    — E1 depth priors must declare source, trap_family, material_stack.

12. `test_e2_depth_prior_requires_full_metadata`
    — E2 depth priors must include technique, uncertainty, extraction.

13. `test_generic_t_eff_rejected`
    — Validator rejects profiles with physical t_eff parameter.

14. `test_calibrated_t_eff_blocked`
    — Validator blocks calibrated t_eff (E3/E4).

15. `test_interface_vs_border_trap_family_required`
    — Depth priors must declare trap_family (interface/border/oxide).

## Non-Goals for First C2 Implementation

- No solver coupling in the first C2 mapping RED/GREEN cycle.
- No quantum confinement solver.
- No calibration claims.
- No device prediction.
- No coherence/fidelity claims.
- No direct modification of existing solver code.

## Expected Module

Future `src/mvp_quantum_materials/charge_mapping.py` (tentative name)
would contain:

- Sheet-source representation (C2-A).
- Volume regularization helper (C2-B, if needed).
- Depth prior registry (C2-C, if needed).
- Validators rejecting C2-D/E.

## Prerequisites

1. ADR-012 must be Accepted.
2. v0.7.0 must be released (documentation-only).
3. Council recommendation must be reviewed by user.
4. RED tests must be created before implementation.
