# v0.8.9 Policy Extension Ratification

## Decision

Human ratifies the minimal extension of `tools/quality_gates/policy.json` for the GREEN module.

## File Authorized

`one_way_poisson_coupling.py`

## Policy Field

`authorized_files` only.

## Reason

The GREEN module name and public contract necessarily include the term "Poisson" because ADR-016 accepted the one-way Poisson coupling strategy. The project scope guardrail treats "Poisson" as conditionally forbidden unless the file is explicitly authorized.

## Scope of Authorization

Allowed:

* Add `one_way_poisson_coupling.py` to `authorized_files`.

Not allowed:

* Remove forbidden terms.
* Remove conditionally forbidden terms.
* Weaken gates.
* Authorize solver runtime.
* Authorize `solve_poisson`.
* Authorize `poisson_solver_2d` imports.
* Authorize physical phi.
* Authorize dummy phi.
* Authorize potential grid.
* Authorize calibration claims.
* Authorize device prediction.
* Authorize paid API.
* Authorize external SDK.
* Authorize goal-like autonomous execution.

## Classification

This is a governance authorization update, not a physical-model validation and not a solver implementation.

## Audit Evidence

The `git diff main -- tools/quality_gates/policy.json` confirms:

* Only the `authorized_files` array was modified.
* Two filenames added: `test_one_way_poisson_coupling_red.py` (already ratified in HDL-015) and `one_way_poisson_coupling.py`.
* No other fields changed.

## Residual Risk

The word "Poisson" in the module name may be misread as runtime coupling. This risk is mitigated by explicit safety validators, no solver imports, no `solve_poisson`, and documentation that the module is metadata-only.

## Review Trigger

Revisit this authorization before any future change that:

* Imports a solver.
* Calls a solver.
* Emits numerical potential output.
* Changes interpretation from demonstrative to physical.
* Changes calibration status.
* Adds runtime behavior.
