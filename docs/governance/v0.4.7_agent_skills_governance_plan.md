> **Date:** 2026-05-13

---

## Purpose

This release formalizes the AI-RSE governance process of the MVP Quantum
Materials project into project-scoped Antigravity Agent Skills. These skills
transform repetitive governance rituals — branch inspection, TDD discipline,
dimensional auditing, scope protection, release management, and report
verification — into reusable, version-controlled playbooks that the agent
can follow consistently across sessions.

## Why Now

The v0.4.6 release stabilized the metadata scaffolding and runtime CSV
integration. Before advancing to Option C (physical charge mapping) or any
new physics, the project needs to consolidate its governance automation:

- **Reduce prompt size.** Current governance prompts are 300+ lines. Skills
  encode the same rules in persistent, reusable documents.
- **Improve consistency.** Skills ensure the same validation sequence is
  followed regardless of session context or prompt wording.
- **Prevent regression.** Skills codify lessons learned (e.g., v0.4.4 test
  count mismatch, tag-before-CI errors) as permanent guardrails.
- **Prepare for complexity.** Option C will involve dimensional conversions,
  charge sign conventions, and physical interpretation — all requiring
  stricter governance automation.

## Installation Scope

All skills are **project-scoped**, installed under:

```
.agent/skills/
```

They are version-controlled in the repository and travel with the codebase.
They are **not** installed globally and do not affect other projects.

## No External Skills

No external skill packages were installed. Specifically:

- No `antigravity-awesome-skills` or similar packages.
- No skills from external registries.
- No dependencies added to `pyproject.toml`.

## No Executable Scripts

This release contains **only Markdown playbooks** (`SKILL.md` files).
No executable scripts, shell commands, or automation hooks are included
in the skills directories.

## Future Evolution

Future releases may add:

- Deterministic validation scripts within skills (via dedicated PR).
- Skill-triggered pre-commit hooks.
- Skill composition for complex multi-phase workflows.

Any such addition will require its own PR review and acceptance gates.
