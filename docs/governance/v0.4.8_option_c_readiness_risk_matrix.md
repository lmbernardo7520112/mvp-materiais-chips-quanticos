## R1 — False Calibration from Literature Constants

- **Severity:** Critical
- **Likelihood:** Medium
- **Description:** Using literature-scaled constants (ε_r, q_e, ε₀) alongside
  toy parameters (D_it, t_eff) could create a false impression of physical
  calibration. Outputs would have SI units but lack physical validity.
- **Mitigation:** `physical_interpretation_allowed` must remain `False` until
  all parameters have evidence tier ≥ T2. The `physics-dimensional-audit`
  skill requires separating proxy from physical quantities. Gate G8 blocks
  Option C; Gate G15 blocks physical φ interpretation.
- **Related Gate:** G8, G15
- **Responsible Skill:** physics-dimensional-audit

## R2 — Incorrect D_it Unit Conversion

- **Severity:** Critical
- **Likelihood:** Medium
- **Description:** The conversion D_it [eV⁻¹·cm⁻²] → D_it_SI [J⁻¹·m⁻²]
  involves two unit changes (energy and area). An error in either factor
  propagates through the entire charge chain (σ_eff, ρ_eff, φ).
- **Mitigation:** The `physics-dimensional-audit` skill requires explicit
  dimensional chain verification. Future TDD RED tests must verify the
  conversion factor (6.242e22) against independent calculation. Gate G9
  blocks implementation until verified.
- **Related Gate:** G9
- **Responsible Skill:** physics-dimensional-audit

---

## R3 — Missing δE_window Convention

- **Severity:** Critical
- **Likelihood:** High
- **Description:** Without a defined energy integration window, N_it = D_it ×
  δE_window is undefined. Choosing the full bandgap (1.12 eV) vs. a kT
  window (0.026 eV at 300K) changes N_it by a factor of ~43. This directly
  scales σ_eff and ρ_eff.
- **Mitigation:** ADR-009 must define δE_window with justification, evidence
  tier, and sensitivity analysis. Gate G12 blocks implementation. The
  `physics-dimensional-audit` skill requires parameter tier classification.
- **Related Gate:** G12
- **Responsible Skill:** physics-dimensional-audit

---

## R4 — Wrong Charge Sign Convention

- **Severity:** Critical
- **Likelihood:** High
- **Description:** Interface traps at Si/SiO₂ can be donor-like (positive when
  empty) or acceptor-like (negative when filled). The net charge depends on
  the Fermi level position relative to the charge neutrality level. Using
  the wrong sign inverts the electrostatic potential.
- **Mitigation:** ADR-009 must document the sign convention with references.
  Future RED tests must verify sign consistency. Gate G10 blocks physical
  charge implementation.
- **Related Gate:** G10
- **Responsible Skill:** physics-dimensional-audit

---

## R5 — t_eff Treated as Physical Thickness Instead of Regularization

- **Severity:** High/Critical
- **Likelihood:** Medium
- **Description:** t_eff converts surface charge density (σ_eff [C/m²]) to
  volume charge density (ρ_eff [C/m³]). It is a regularization parameter,
  not the physical oxide thickness. Treating it as physical would create
  incorrect ρ_eff magnitudes.
- **Mitigation:** Documentation must explicitly label t_eff as a regularization
  parameter with tier TX or T3. The `physics-dimensional-audit` skill
  requires identifying parameters as toy/demonstrative/literature/calibrated.
  Gate G11 blocks implementation.
- **Related Gate:** G11
- **Responsible Skill:** physics-dimensional-audit

---

## R6 — normalized_2d Mistaken for Physical Geometry

- **Severity:** High
- **Likelihood:** Medium
- **Description:** The current solver uses `geometry_mode = normalized_2d` with
  domain [0,1]×[0,1]. Physical ε substitution on a normalized domain produces
  φ values that are neither dimensionless nor in volts — they are in an
  inconsistent unit system.
- **Mitigation:** Option C must define a `physical_2d` geometry mode with real
  dimensions (meters). The `scope-guardrails` skill blocks geometry changes
  without ADR. Gate G14 blocks solver changes.
- **Related Gate:** G14
- **Responsible Skill:** scope-guardrails

---

## R7 — φ Interpreted as Physical Voltage Prematurely

- **Severity:** Critical
- **Likelihood:** Medium
- **Description:** If any upstream term (D_it, δE_window, sign, t_eff, ε, geometry)
  is incorrect or toy, φ cannot be interpreted as a physical voltage. Premature
  interpretation would constitute overclaim.
- **Mitigation:** `physical_interpretation_allowed` must remain `False` until
  all upstream terms are verified. Gate G15 blocks this. The
  `physics-dimensional-audit` skill explicitly prohibits physical φ
  interpretation in demonstrative mode.
- **Related Gate:** G15
- **Responsible Skill:** physics-dimensional-audit

---

## R8 — Surface Charge Incorrectly Smeared into Volume

- **Severity:** High
- **Likelihood:** Medium
- **Description:** The σ_eff → ρ_eff conversion via t_eff is a volume-smearing
  approximation. The alternative is a surface charge boundary condition.
  Choosing the wrong model changes the Poisson formulation fundamentally.
- **Mitigation:** ADR-009 must specify whether Option C uses surface-mode
  (boundary condition) or volume-smearing-mode (source term). The current
  `geometry_mode` enum supports this distinction but the solver does not
  implement it. Gate G10 blocks implementation.
- **Related Gate:** G10
- **Responsible Skill:** physics-dimensional-audit

---

## R9 — ε Substitution Creates False Physical Legitimacy

- **Severity:** High
- **Likelihood:** Low
- **Description:** Replacing ε = 1 with ε = ε_r × ε₀ in the solver, without
  simultaneously fixing all other terms (geometry, source, BCs), creates a
  partially physical model that is harder to interpret than a fully
  demonstrative one.
- **Mitigation:** ε substitution must be atomic with all other physical terms.
  Gate G13 blocks ε substitution. The `scope-guardrails` skill requires
  all-or-nothing physical mode transitions.
- **Related Gate:** G13
- **Responsible Skill:** physics-dimensional-audit

---

## R10 — Skills Become Stale and Allow Implementation Without ADR

- **Severity:** High
- **Likelihood:** Medium
- **Description:** If the project advances past v0.4.x without updating skills,
  the `scope-guardrails` skill may reference outdated roadmap assumptions,
  allowing implementation that should still be blocked.
- **Mitigation:** Skills must be reviewed and updated as part of any ADR that
  changes the project scope. Gate G4 requires skills to be read and
  referenced. The `report-auditor` skill checks for stale assumptions.
- **Related Gate:** G4
- **Responsible Skill:** scope-guardrails

---

## R11 — Quality Gates Allow Terminology but Not Semantics

- **Severity:** Medium/High
- **Likelihood:** Medium
- **Description:** The quality gates check for forbidden terms and file
  modifications, but cannot verify whether an equation is dimensionally
  correct or whether a parameter tier is appropriate.
- **Mitigation:** The `physics-dimensional-audit` skill compensates by
  requiring manual dimensional verification. Future deterministic scripts
  in skills could add automated dimensional checks. Gate G16 verifies
  quality gates pass.
- **Related Gate:** G16
- **Responsible Skill:** ai-rse-gateops

---

## R12 — Future Tests Validate Labels but Not Dimensional Consistency

- **Severity:** High
- **Likelihood:** Medium
- **Description:** Tests like `test_runtime_scale_metadata.py` verify that
  metadata labels exist and have correct values, but do not verify that
  the underlying computations are dimensionally consistent.
- **Mitigation:** Future Option C TDD RED tests must include dimensional
  consistency checks (e.g., verifying that ρ_eff has units [C/m³] by
  checking magnitudes against analytical expectations). The
  `tdd-red-green-release` skill requires comprehensive RED specifications.
- **Related Gate:** G17
- **Responsible Skill:** tdd-red-green-release
