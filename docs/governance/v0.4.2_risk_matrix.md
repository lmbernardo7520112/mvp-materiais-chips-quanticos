# Risk Matrix — v0.4.2 SI Unit Conversion & Parameter Scale Audit

> **Date:** 2026-05-11
> **ADR:** ADR-008 (Proposed)

---

## Risk Register

### R1 — False Calibration from SI Units

| Attribute | Value |
|-----------|-------|
| **Severity** | **Critical** |
| **Likelihood** | Medium |
| **Description** | Introducing SI values (ε_r, D_it) creates output (φ in volts) that appears physically meaningful. External readers may interpret this as calibrated device prediction. |
| **Mitigation** | Separate "demonstrative" and "literature-scaled" modes. Default remains demonstrative. All literature-scaled outputs carry explicit disclaimers. Quality gates scan for forbidden calibration language. |
| **Gate** | G6 (Overclaim Prevention) |
| **Owner** | GOV (Model Governance Specialist) |

---

### R2 — Confusing Normalized Domain with Physical Geometry

| Attribute | Value |
|-----------|-------|
| **Severity** | **Critical** |
| **Likelihood** | Medium |
| **Description** | The current domain [0, 1] × [0, 1] is normalized. If SI units are introduced for ε and ρ_eff without changing the domain, x and y become ambiguous: are they in meters? nanometers? normalized? |
| **Mitigation** | Domain metadata must explicitly state `is_physical: false` or provide Lx, Ly with units. Output plots must label axes as "normalized" or with physical units. |
| **Gate** | G7 (Domain-Unit Decision) |
| **Owner** | RSE (Research Software Engineer) |

---

### R3 — Using D_it Without Energy Integration Convention

| Attribute | Value |
|-----------|-------|
| **Severity** | **High** |
| **Likelihood** | Medium |
| **Description** | D_it [eV⁻¹ cm⁻²] is an energy-resolved density. Computing σ_eff = q · D_it · ΔE requires specifying the energy window ΔE. Using D_it without ΔE produces a surface charge with undefined physical meaning. In real devices, D_it(E) varies by orders of magnitude across the bandgap. |
| **Mitigation** | ADR-008 documents the requirement for explicit ΔE specification. Option C (full charge closure) is deferred until ΔE convention is established. |
| **Gate** | G4 (Unit Chain Documented) |
| **Owner** | SDP (Semiconductor Device Physicist) |

---

### R4 — Surface-to-Volume Conversion via Arbitrary t_eff

| Attribute | Value |
|-----------|-------|
| **Severity** | **High** |
| **Likelihood** | High |
| **Description** | The conversion σ_eff → ρ_eff = σ_eff / t_eff requires a physical thickness t_eff. Using t_eff = 1.0 (current code) or t_eff = 1 nm (common but arbitrary) creates false precision. The choice of t_eff changes ρ_eff by orders of magnitude. |
| **Mitigation** | t_eff remains demonstrative (T0/TX) until device-specific justification exists. Option C deferred. ADR-008 documents t_eff as a critical parameter requiring justification. |
| **Gate** | G4 (Unit Chain Documented), G8 (Future Implementation Requirements) |
| **Owner** | SDP / CMS |

---

### R5 — ε_r · ε₀ Changes φ Magnitude Dramatically

| Attribute | Value |
|-----------|-------|
| **Severity** | **High** |
| **Likelihood** | High (certain if implemented) |
| **Description** | Replacing ε = 1.0 with ε = ε_r · ε₀ ≈ 10⁻¹⁰ changes the Poisson equation coefficient by 10 orders of magnitude. The resulting φ will be ~10¹⁰ times larger (for same ρ_eff), completely changing the numerical character of the solution. |
| **Mitigation** | Option B introduces ε_r · ε₀ only, keeping ρ_eff demonstrative. Solver convergence and tolerance must be re-validated. Order-of-magnitude sanity checks required. |
| **Gate** | G10 (Validation Green) |
| **Owner** | NMS (Numerical Methods Specialist) |

---

### R6 — Mixing Si, SiO₂, and SiGe Parameter Contexts

| Attribute | Value |
|-----------|-------|
| **Severity** | **High** |
| **Likelihood** | Low |
| **Description** | Different material stacks have different ε_r, D_it ranges, and defect species. Using ε_r(Si) = 11.7 with D_it values from SiO₂ literature or SiGe heterostructures without explicit stack documentation creates physical inconsistency. |
| **Mitigation** | Parameter registry classifies each parameter by `material_stack`. ADR-008 restricts scope to silicon/CMOS spin-qubit context only. No SiGe, GaN, or HfO₂ parameters. |
| **Gate** | G6 (Overclaim Prevention) |
| **Owner** | CMS (Computational Materials Scientist) |

---

### R7 — Treating Literature Range as Device-Specific Calibration

| Attribute | Value |
|-----------|-------|
| **Severity** | **Critical** |
| **Likelihood** | Medium |
| **Description** | D_it = 10¹⁰–10¹² eV⁻¹cm⁻² is a literature range spanning 2 orders of magnitude. Choosing a point value (e.g., 10¹¹) and using it as if it were device-specific calibration is false precision. The chosen value determines φ linearly. |
| **Mitigation** | Any literature value used must be labeled as "range midpoint" or "order-of-magnitude guide," not "device calibration." Outputs must state the range uncertainty. |
| **Gate** | G5 (Demonstrative vs Literature-Scaled Distinction), G6 (Overclaim Prevention) |
| **Owner** | GOV |

---

## Residual Risk Assessment

All identified risks are mitigated by the documentation-only nature of v0.4.2.
No code changes are made, so no risk can materialize in the codebase during
this release.

The primary residual risks for **future implementation** are:

1. **R1 (False calibration):** Remains critical and must be addressed by
   implementation gates in the Option B PR.
2. **R4 (Arbitrary t_eff):** Deferred to Option C; not a risk for Option B.
3. **R5 (ε magnitude change):** Certain to occur in Option B; mitigated by
   solver re-validation and φ sanity checks.
4. **R7 (Literature range as calibration):** Ongoing risk for any
   literature-scaled mode; mitigated by labeling conventions.
