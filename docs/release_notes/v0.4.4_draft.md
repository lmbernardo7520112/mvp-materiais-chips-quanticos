# v0.4.4 — SI Constants Scaffolding (Option B)

> **Date:** 2026-05-11
> **ADR:** ADR-008 (Accepted)
> **Branch:** `feature/v0.4.4-si-constants-scaffolding`

## Summary

v0.4.4 implements the **Option B** minimal scaffolding from ADR-008:
literature-scaled SI constants and scale mode metadata infrastructure.

This release adds the dimensional infrastructure that future SI-unit
work can build upon, while preserving the demonstrative mode as the
default and explicitly blocking any physical interpretation of φ.

## New Modules

| Module | Purpose |
|--------|---------|
| `src/mvp_quantum_materials/units.py` | SI constants (ε₀, q) and reference relative permittivity lookups |
| `src/mvp_quantum_materials/scale_modes.py` | Scale mode, geometry mode, and potential interpretation metadata |

## What This Release Does

- Makes `EPSILON_0` (CONST_DERIVED/CODATA) and `ELEMENTARY_CHARGE` (CONST_EXACT) available.
- Provides `relative_permittivity("Si")` → 11.7 and `relative_permittivity("SiO2")` → 3.9.
- Provides `absolute_permittivity(epsilon_r)` → ε_r · ε₀.
- Defines `ScaleMode.DEMONSTRATIVE` (default) and `ScaleMode.LITERATURE_SCALED_CONSTANTS`.
- Defines `GeometryMode.NORMALIZED_2D` (default).
- Defines `PotentialInterpretation.DEMONSTRATIVE` (default) and `DIMENSIONAL_SCAFFOLDING`.
- Defines `ScaleMetadata` dataclass with `physical_interpretation_allowed()` → `False` by default.

## What This Release Does NOT Do

- Does NOT implement D_it → D_it_SI conversion.
- Does NOT implement σ_eff physical computation.
- Does NOT implement ρ_eff physical or calibrated computation.
- Does NOT implement t_eff physical justification or use.
- Does NOT implement delta_E_window calculation.
- Does NOT alter the electrostatic solver.
- Does NOT alter the effective charge module.
- Does NOT alter any script or pipeline.
- Does NOT make φ physically interpretable.
- Does NOT calibrate the model.
- Does NOT provide device-level prediction.
