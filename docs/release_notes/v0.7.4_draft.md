# v0.7.4 — C2 Charge Mapping Demo & Sanity Checks

## Summary

Adds isolated demonstration artifacts for C2 charge mapping.

## Added

- `scripts/run_c2_charge_mapping_demo.py`
- `tests/test_c2_charge_mapping_demo.py`
- `results/csv/c2_charge_mapping_demo.csv`
- `results/figures/c2_charge_conservation.png`
- `results/figures/c2_l_reg_sensitivity.png`

## Demonstrated

- interface sheet source;
- conservative volume regularization;
- charge conservation (relative error ≤ 1e-12);
- l_reg sensitivity (rho ∝ 1/l_reg);
- sign preservation (positive and negative sigma);
- safe metadata flags (not_calibrated, numerical, no solver coupling).

## Not Implemented

- no solver coupling;
- no Poisson runtime;
- no physical phi;
- no quantum confinement solver;
- no C3;
- no calibration claims;
- no device prediction.

## Dependency Decision

The first CI run failed because tests imported pandas, which is not a project dependency.
The tests were rewritten to use Python's stdlib csv module.
This was accepted because v0.7.4 requires only small deterministic CSV validation.
No new dependency was added.
