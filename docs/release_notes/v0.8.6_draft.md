# Release Notes — v0.8.6

## ADR-016 Acceptance Review

**Type:** Documentation-only release.

### Summary
v0.8.6 formalizes the acceptance of ADR-016 (One-way Poisson coupling strategy). This decision authorizes a future RED planning phase for coupling C3 logic to a demonstrative Poisson solver. It does **not** authorize any implementation.

### Key Confirmations
* **ADR-016 is now Accepted.**
* **No Implementation:** Zero changes to `src/`, `tests/`, or `scripts/`.
* **No Poisson Runtime:** The electrostatic equation remains uncoupled and unexecuted.
* **No Physical Phi:** No potential fields are calculated or output.
* **Non-Authorization Clause Enforced:** All constraints on calibration and device prediction remain fully active.
* **BudgetOps & Usage Ledger:** Mandatory validation checks passed perfectly.

### Next Steps
The next authorized step is a future RED planning phase for the one-way demonstrative Poisson coupling, strictly following ADR-016 guidelines. No code can be written until that RED phase is formally accepted.
