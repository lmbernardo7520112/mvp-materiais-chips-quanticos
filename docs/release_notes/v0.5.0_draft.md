> **Date:** 2026-05-14
> **Branch:** `feature/v0.5.0-c1-surface-density-bookkeeping`
> **ADR:** ADR-009 Accepted

## Summary

Implements the first physics-forward C1 bookkeeping module authorized by
ADR-009 Accepted. This is the first new physics code since v0.4.6.

The release implements only:

```
D_it [eV⁻¹·cm⁻²] → D_it_SI [J⁻¹·m⁻²] → N_it [m⁻²] → σ_eff [C/m²]
```

It does not couple charge to the PDE solver and does not compute ρ_eff.

## Physics Scope

C1 computes surface-density bookkeeping only:

- D_it [eV⁻¹·cm⁻²] — input
- D_it_SI [J⁻¹·m⁻²] — converted via `1e4 / q_e`
- N_it [m⁻²] — `D_it_SI × δE_window`
- σ_eff [C/m²] — `s_charge × q_e × N_it × f_occ`

## Not Implemented

- No ρ_eff.
- No t_eff.
- No volume regularization.
- No solver coupling.
- No ε substitution.
- No physical φ interpretation.
- No C2/C3.
- No runtime `option_c_enabled = True`.
