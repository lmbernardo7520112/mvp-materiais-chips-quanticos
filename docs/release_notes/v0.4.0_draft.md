# Release Notes — v0.4.0 (Draft)

> **Date:** 2026-05-10
> **Branch:** `feature/v0.4-poisson-bridge-core`
> **Status:** Pending merge and tag

## Summary

v0.4 implements the **Demonstrative Poisson Bridge**, connecting the defect
reaction-diffusion output (C_def) to a simplified electrostatic potential (φ)
via a Boussinesq-inspired effective charge closure (ADR-007, Option C-B).

This release is **strictly demonstrative**. It does not claim calibrated
physical predictions, device-level accuracy, or semiconductor design capability.

## New Modules

### `effective_charge.py`

- `EffectiveChargeParams` — dataclass with demonstrative, uncalibrated defaults.
- `compute_theta(T)` — dimensionless thermal activation parameter.
- `h_cdef(C_def, C_sat)` — dimensionless normalized defect concentration in [0, 1].
- `compute_delta_sigma_eff(...)` — mean-subtracted surface charge proxy.
- `convert_sigma_to_rho(delta_sigma, t_eff)` — surface-to-volume conversion.
- `compute_delta_rho_eff(C_def, T, params)` — main entry point for δρ_eff.

### `poisson_solver_2d.py`

- `PoissonSolveResult` — dataclass exposing phi, iterations, residual, converged.
- `solve_poisson_2d_demonstrative(rho, dx, dy, epsilon, ...)` — Jacobi iterative
  solver for ∇²φ = −ρ/ε with homogeneous Dirichlet boundaries.

### `run_poisson_bridge.py`

- Orchestration script connecting effective_charge → Poisson solver → artifacts.
- Generates `poisson_bridge_potential.png` and `poisson_bridge_metrics.csv`.
- Integrated into `generate_all_results.py`.

## Mathematical Scope

| Property | Value |
|----------|-------|
| Equation | ∇²φ = −ρ/ε (homogeneous ε) |
| Boundary | Homogeneous Dirichlet (φ=0) |
| Method | Vectorized Jacobi iteration |
| ε | Constant (1.0, demonstrative) |
| C_def | Dimensionless, in [0, 1] |
| δρ_eff | Mean-subtracted, uncalibrated |
| Self-consistency | None (no φ → occupancy feedback) |

## Explicit Non-Claims

> [!CAUTION]
> v0.4 does **not**:
> - Solve nonlinear Poisson
> - Implement Neumann or periodic boundaries
> - Use spatially varying ε(x,y)
> - Implement self-consistent φ → trap occupancy → ρ_eff loops
> - Solve Schrödinger
> - Interface with TCAD/QTCAD
> - Predict device-level charge noise or qubit fidelity
> - Use calibrated physical parameters
