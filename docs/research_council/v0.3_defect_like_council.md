## 4. Individual Assessments

### 4.1 Chair — Computational Materials Science

The v0.3 step is scientifically justified. The project has earned the
right to introduce a defect-like variable through 2 releases of
numerical infrastructure. The key risk is overclaim: C_def must remain
explicitly adimensional and demonstrative. The reaction-diffusion
formalism from crystal growth literature (Dornberger, Brown, Voronkov)
provides a mature equation structure that can be simplified for our
educational pipeline.

### 4.2 Solidification / Heat-Mass Transport

The thermal solver is stable and convergent. The natural next step is
coupling a scalar variable to the thermal field. The Arrhenius diffusivity
is already implemented in 1D (diffusion_solver.py). Extending to 2D
with generation/recombination terms is methodologically sound. The
solver should be explicit Euler initially, with stability guard rails.

### 4.3 Crystal Growth / Point Defects in Silicon

The full V/I coupled system (Dornberger 2001) requires 10+ parameters
per species (D₀, E_m, E_f, k_rec, ...) and coupled equations. This is
too complex for v0.3. A single effective C_def that captures the
*qualitative behavior* — generation in a thermal window, diffusion,
recombination/relaxation, freeze-in — is scientifically defensible as
a reduced model. The Voronkov v/G framework shows that even simple
criteria can capture essential physics.

### 4.4 Si/SiO₂ Interfaces / Charge Noise

Interface traps (Pb centers, dangling bonds) are a major noise source
for MOS qubits (Martinez & Niquet 2022). C_def can be interpreted as
a trap-like/charge-disorder-like variable that, in future versions,
maps to ρ_eff. This is the correct abstraction level for v0.3: we
model the *spatial distribution* of disorder, not its microscopic origin.

### 4.5 Spin Qubits / Quantum Dots

Charge noise limits qubit coherence and gate fidelity (Culcer 2009,
Shehata 2023). The spatial distribution of charge disorder determines
the electrostatic landscape of the quantum dot. A process model that
produces C_def(x,y) as input for future Poisson is exactly what the
device community needs but rarely connects to. This is the gap.

### 4.6 TCAD / Poisson-Schrödinger

QTCAD and Sentaurus solve Poisson-Schrödinger with background charges.
The background charge distribution is typically assumed or extracted
from process simulation. Our v0.3 produces exactly that: a spatially
resolved C_def that can be converted to ρ_eff in v0.4. The interface
is clean: C_def(x,y) → q_eff · C_def → ρ_eff → Poisson source term.

### 4.7 Reaction-Diffusion Methods

The equation ∂C/∂t = ∇·(D(T)∇C) + G(T) − R(T,C) is a standard
reaction-diffusion PDE. For explicit Euler, the CFL stability condition
must include the reaction terms. With bounded G and linear R, the
system is well-behaved. The key numerical concern is ensuring
C_def remains non-negative and bounded (if bounded model chosen).

### 4.8 Software Governance / AcademiaFlow

The v0.3 specification must follow the same governance as v0.2: ADR
before implementation, acceptance gates, TDD, CI/CD, coverage gate.
The additive-only policy must be preserved — v0.1/v0.2 solvers must
remain untouched. New modules for C_def solver, new tests, new scripts.

## 5. Candidate Models

### Model A — Full V/I System

```
∂C_V/∂t = ∇·(D_V∇C_V) − k_rec(C_V·C_I − C_V^eq·C_I^eq)
∂C_I/∂t = ∇·(D_I∇C_I) − k_rec(C_V·C_I − C_I^eq·C_V^eq)
```

- **Pros:** Closest to literature (Dornberger, Brown)
- **Cons:** 10+ parameters, coupled equations, requires equilibrium
  concentrations, too complex for v0.3 scope
- **Verdict:** REJECTED for v0.3. Candidate for v0.5+ if calibration
  data becomes available.

### Model B — Effective Interface-Trap / Charge-Disorder

```
∂C_trap/∂t = G_trap(V_gate, T) − R_trap(C_trap)
```

- **Pros:** Directly relevant to device performance
- **Cons:** Requires gate voltage model (not in v0.3), device-level
  abstraction inappropriate before Poisson
- **Verdict:** REJECTED for v0.3. Belongs to v0.4/v0.5.

### Model C — Generic Defect-like Scalar C_def ✅

```
∂C_def/∂t = ∇·(D(T)∇C_def) + G(T)(1 − C_def/C_sat) − R(T)·C_def
```

- **Pros:**
  - Single variable, single equation
  - Standard reaction-diffusion form
  - Arrhenius D(T) already demonstrated in 1D
  - Bounded if C_sat > 0 and R > 0
  - Naturally produces spatial heterogeneity from thermal field
  - Clean interface to future ρ_eff (v0.4)
  - 6–8 parameters (manageable, documentable)
  - Compatible with explicit Euler + CFL guard
- **Cons:**
  - Not calibrated to any specific defect type
  - Cannot distinguish vacancies from interstitials
  - Results are qualitative only
- **Verdict:** RECOMMENDED for v0.3.

### Model D — Defer v0.3

- **Pros:** Zero risk of overclaim
- **Cons:** Project stagnates at "thermal-only" infrastructure;
  loses momentum toward process-to-device bridge
- **Verdict:** REJECTED. The project has sufficient foundation.

## 7. Recommended Formulation

### 7.1 Governing Equation

```
∂C_def/∂t = ∇·(D(T)∇C_def) + G(T)·(1 − C_def/C_sat) − R(T)·C_def
```

### 7.2 Diffusivity

```
D(T) = D₀ · exp(−E_D / (k_B · T))
```

Arrhenius form, consistent with diffusion_solver.py (1D) and crystal
growth literature.

### 7.3 Generation Term

**Recommended: thermal window Gaussian**

```
G(T) = A_G · exp(−(T − T_G)² / (2·σ_G²))
```

Rationale: defects are generated preferentially in a critical
temperature window during cooling. The Gaussian captures this
without requiring detailed nucleation theory.

### 7.4 Recombination/Relaxation Term

**Recommended: linear relaxation with Arrhenius rate**

```
R(T) = A_R · exp(−E_R / (k_B · T))
```

Applied as R(T)·C_def (first-order kinetics). Simple, bounded,
well-behaved numerically. Consistent with recombination models in
crystal growth.

### 7.5 Saturation

C_sat bounds C_def from above. With the logistic term (1 − C_def/C_sat),
C_def naturally stays in [0, C_sat]. Setting C_sat = 1.0 makes C_def
a normalized, adimensional field in [0, 1].

### 7.6 Boundary Conditions

- **Neumann no-flux** (∂C_def/∂n = 0) on all boundaries
- Rationale: defects generated internally by thermal field;
  no external source/sink at boundaries

### 7.7 Initial Condition

- C_def(x,y,0) = 0 (pristine material)
- Alternative: C_def(x,y,0) = C_background (small uniform seed)

### 7.8 Coupling

- **One-way:** T(x,y,t) drives C_def(x,y,t) via D(T), G(T), R(T)
- T field is computed first (thermal_solver_2d), then C_def evolves
  on the same or interpolated temperature history
- **No back-coupling** of C_def on T (justified: defect concentrations
  are too low to affect thermal conductivity at this scale)

## 8. What CANNOT Be Inferred from v0.3

- ❌ Real defect density in silicon
- ❌ Real trap concentration at Si/SiO₂
- ❌ Charge noise magnitude
- ❌ Qubit coherence or dephasing
- ❌ Gate fidelity
- ❌ Wafer quality
- ❌ Experimental validation
- ❌ Electrostatic potential (requires Poisson, v0.4)
- ❌ Confinement levels (requires Schrödinger, v0.5)

## 9. Council Decision

**APPROVED: Model C — Generic defect-like scalar C_def**

With the following guardrails:
1. C_def is adimensional, bounded in [0, 1]
2. All parameters are toy/demonstrative
3. No parameter presented as physically calibrated without source
4. Maurand et al. cited ONLY as device-target anchor
5. All figures must carry "demonstrative — not calibrated" disclaimer
6. Every metric must state whether it is a proxy
7. Additive-only: v0.1/v0.2 solvers untouched
8. No Poisson in v0.3
9. No Schrödinger in v0.3
10. ADR-006 required before implementation
