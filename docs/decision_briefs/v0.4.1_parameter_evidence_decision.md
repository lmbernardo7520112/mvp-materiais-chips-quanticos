# Parameter Evidence Decision Brief — v0.4.1

> **Date:** 2026-05-10
> **Decision:** Document parameter evidence tiers; do NOT calibrate
> **ADR:** ADR-007 (Accepted, v0.4 scope)

## Context

The v0.4 Poisson Bridge uses multiple parameters across three subsystems:
(1) defect reaction-diffusion, (2) effective charge closure, and
(3) Poisson solver. The v0.4.1 curation audits these parameters against
primary literature to clarify which values are toy, which are literature-
consistent, and which would require device-specific calibration.

## Decision

> [!IMPORTANT]
> **No parameter is promoted to "calibrated" in v0.4.1.**
> This release classifies evidence tiers only. Calibration requires
> a dedicated ADR and experimental validation.

## Evidence Tier Assignments

### Promotions (with justification)

| Parameter | From | To | Justification |
|-----------|------|----|---------------|
| D0_def | T0 | T1 | Current value (10⁻⁸ m²/s) falls within the literature range for vacancy-mediated diffusion (Bracht 1998). |
| N_ref | TX | T1 | Could be anchored to D_it ranges if unit conversion is implemented. Currently remains dimensionless. |

### Confirmed TX (no promotion possible)

| Parameter | Tier | Reason |
|-----------|------|--------|
| lambda_T | TX | No physical anchor. Demonstrative coupling coefficient. |
| lambda_C | TX | No physical anchor. Demonstrative coupling coefficient. |
| C_sat | TX | Dimensionless normalization. C_def is not physical. |
| epsilon | TX | Solver uses 1.0; physical would be ε_r·ε₀. |

### Confirmed T0 (remain toy)

| Parameter | Tier | Reason |
|-----------|------|--------|
| G0 | T0 | No experimental anchor for functional form. |
| R0 | T0 | No experimental anchor. |

### New T2 Candidates (not yet used in code)

| Parameter | Tier | Range | Source |
|-----------|------|-------|--------|
| ε_r(Si) | T2 | 11.4–11.9 | Sze & Ng 2007 |

### T3 Candidates (calibration required)

| Parameter | Tier | Reason |

| t_eff | T3 | Device-specific geometric parameter. |

## Risks of Premature Promotion

> [!CAUTION]
> Promoting parameters to T2/T3 without implementing unit conversion
> in the code creates a false sense of calibration. The MVP must either:
> (a) use demonstrative values with TX/T0 labels, or
> (b) implement full SI unit handling before using T2 values.

## Recommendation

1. Maintain all active code parameters at their current toy/demonstrative values.
2. Record literature ranges in the registry for future reference.
3. If a future version implements unit conversion and physical ε, create
   an ADR to authorize the transition from demonstrative to literature-anchored.
