# v0.7.4 CSV vs pandas Dependency Decision

## Context

The first PR #43 CI run failed because tests imported pandas, but pandas is not a project dependency.

## Decision

Keep Python stdlib csv for v0.7.4.

Do not add pandas.

## Rationale

- v0.7.4 uses small deterministic CSV artifacts.
- The demo requires only simple row/column validation.
- csv stdlib is sufficient.
- Adding pandas would introduce unnecessary dependency surface.
- Avoids dependency creep.
- Keeps CI minimal and reproducible.
- Maintains consistency with earlier project CSV artifact policy.

## When pandas may be reconsidered

Only in a future dedicated dependency ADR/PR if the project requires:

- complex tabular analysis;
- groupby/aggregation;
- joins;
- larger datasets;
- DataFrame-based public API;
- notebook/data-science workflow;
- statistical post-processing beyond stdlib.

## Scope

This decision does not change C2 physics.
This decision does not alter c2_charge_mapping.py.
This decision does not authorize solver coupling.
This decision does not authorize calibration claims.
