# v0.5.6 No Demonstrative Evidence Profiles Brief

## Core Question

Should E0 demonstrative profiles remain admissible for future C1 D_it(E)
implementation?

## Decision

**No.**

E0 must be removed as an operational evidence level for model-facing
D_it(E) profiles. Synthetic profiles are allowed only as **S0 TEST_ONLY**
fixtures for mathematical verification inside test suites.

## Rationale

1. **C1 is now literature-positioned and ADR-governed.** After v0.5.2
   (literature scale benchmark) and v0.5.4 (evidence taxonomy), the project
   has established a scientific baseline that demonstrative profiles would
   undermine.

2. **Future D_it(E) profiles must be evidence-informed.** The ADR-010
   acceptance note (v0.5.5) authorizes only Option B (literature-informed)
   and Option C (E2 experimental prior). E0 demonstrative profiles are
   inconsistent with this accepted direction.

3. **Demonstrative model-facing profiles weaken scientific positioning.**
   A profile labeled "E0 demonstrative" could appear in benchmark CSVs or
   release artifacts, creating ambiguity about whether the model produces
   meaningful physical output or arbitrary values.

4. **Synthetic fixtures remain necessary for testing.** Mathematical
   verification (convergence, monotonicity, dimensional consistency) requires
   synthetic profiles, but these must be clearly labeled as S0 TEST_ONLY
   and confined to test suites.

## New Evidence Level Rules

| Level | Name | Operational? | Where Allowed |
|-------|------|-------------|---------------|
| S0 | TEST_ONLY | ❌ No | Tests only (fixtures) |
| E0 | Demonstrative | ❌ **Deprecated** | Not allowed |
| E1 | Literature-informed | ✅ Yes (default) | Model + artifacts |
| E2 | Experimental-profile prior | ✅ Conditional | Model + artifacts (with metadata) |
| E3 | Same-stack experimental | ❌ Blocked | Requires future ADR |
| E4 | Calibration-grade | ❌ Blocked | Requires future ADR |

## What S0 TEST_ONLY Fixtures May Do

- Provide known-answer inputs for numerical verification.
- Test convergence, monotonicity, symmetry, and dimensional consistency.
- Validate backward compatibility with P0 (single-bin = constant D_it).

## What S0 TEST_ONLY Fixtures Must NOT Do

- Appear in release artifacts as scientific scenarios.
- Be exported in benchmark CSVs as model profiles.
- Be used as default profiles in scripts or demonstrations.
- Be described as evidence-informed.
- Support calibration or device prediction claims.

## Still Blocked

- E3/E4 calibration-grade evidence.
- ρ_eff computation.
- t_eff computation.
- Solver coupling.
- Physical φ interpretation.
- C2/C3 features.
- Calibration claims.
