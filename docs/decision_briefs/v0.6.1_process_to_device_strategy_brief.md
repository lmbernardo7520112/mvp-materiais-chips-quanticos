# v0.6.1 Process-to-Device Strategy Brief

## Core Question

How should the project evolve from C1 energy-distribution bookkeeping
toward a silicon/CMOS qubit process-to-device demonstrator?

## Strategic Options

### Option A — Continue C1 profile refinement only

**Pros:**
- Low risk.
- Improves input data quality.
- No new physics coupling needed.

**Cons:**
- Delays device trajectory indefinitely.
- Project may become a permanent bookkeeping tool.
- Does not advance toward Maurand-inspired goal.

### Option B — Jump directly to Schrödinger

**Pros:**
- Attractive qubit-facing result.
- Skips intermediate layers.

**Cons:**
- Physically premature: no ρ_eff defined.
- No Poisson coupling: V_conf has no physical basis.
- High overclaim risk: Schrödinger without electrostatic context.
- Violates ADR-005 layer ordering.

### Option C — Proceed layer-by-layer process-to-device ✅

**Pros:**
- Physically ordered: each layer feeds the next.
- Auditable: each layer has its own ADR and TDD cycle.
- Compatible with governance (AI-RSE, TDD, skills).
- Preserves scientific credibility.
- Allows course corrections at each layer boundary.

**Cons:**
- Slower: requires multiple ADRs and release cycles.
- More documentation overhead.

### Option D — Attempt calibrated qubit simulation

**Pros:**
- Maximum ambition.

**Cons:**
- Impossible without device/process data from fabrication.
- Violates ADR-002 (no coherence prediction).
- Violates all current scope guardrails.
- Would require TCAD-level infrastructure.

## Recommendation

**Option C: layer-by-layer process-to-device.**

This is the only option that:
1. Preserves physical ordering.
2. Maintains governance integrity.
3. Advances toward the Maurand-inspired goal.
4. Allows honest assessment at each stage.

## Decision

The project should advance layer-by-layer:

```
C1 profile library (v0.6.2)
  → C2 ADR (v0.7.0)
    → Poisson coupling (v0.8.0)
      → Schrödinger confinement (v0.9.0)
        → Integrated demonstrator (v1.0)
```
