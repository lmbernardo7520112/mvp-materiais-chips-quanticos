## A — Parallel with Solidification

### The Boussinesq Approximation

In solidification modeling, the full density ρ depends on temperature T and
solute composition c. The Boussinesq approximation linearizes this:

```
ρ(T,c) ≈ ρ₀ [1 − βT(T − Tref) + βC(c − cref)]
```

The perturbation from the reference state drives buoyancy:

```
δρ = ρ − ρ₀ ≈ ρ₀ [−βT ΔT + βC Δc]

Buoyancy force: f_b = δρ · g
```

**Key principles:**
1. **ρ₀** is a reference density; the full density is never computed.
2. **βT, βC** are sensitivity coefficients (expansion coefficients).
3. Only **perturbations** δρ appear in the governing equations.
4. The approximation is valid for |δρ/ρ₀| ≪ 1.

This strategy avoids catastrophic cancellation from subtracting nearly
equal large numbers and isolates the physically meaningful spatial
variation.

### Why This Matters for the MVP

The MVP faces an analogous problem: fields (T, C_def) must be converted
into a source term for a governing equation (Poisson). Applying the full
Boussinesq philosophy means:

- Define a **reference state** (spatial mean or uniform background).
- Compute only the **perturbation** of charge density.
- Feed the perturbation into Poisson as the source term.

## B — Analogy in the Current Project

### What T and C_def Should NOT Generate

T and C_def must **not** generate:
- Mass density (wrong physics).
- Absolute charge density without reference subtraction.
- Calibrated device-level predictions.

### What T and C_def CAN Generate (in v0.4)

T and C_def can perturb an effective interface trap density, which in turn
produces a charge perturbation:

### Proposed Closure — Option C-B

**Step 1: Normalized fields**

```
θ(x,y) = [T(x,y) − T_ref] / ΔT_ref
```

where T_ref = spatial mean of T, ΔT_ref = characteristic temperature scale.

**Step 2: Bounded proxy function**

```
h(C_def) = C_def / C_sat
```

h ∈ [0, 1], monotonic, bounded. Other choices (tanh, sigmoid) are possible
but C_def/C_sat is the simplest given C_sat = 1.

**Step 3: Effective trap density**

```
D_it_eff(x,y) = D_it_ref · [1 + λT·θ(x,y) + λC·h(C_def(x,y))]
```

where:
- D_it_ref [cm⁻² eV⁻¹] = reference interface trap density (T3 tier).
- λT = thermal sensitivity coefficient (dimensionless, T0/demonstrative).
- λC = defect sensitivity coefficient (dimensionless, T0/demonstrative).
- |λT|, |λC| ≪ 1 for the linearization to hold.

**Step 4: Trap occupancy integral**

```
I(T) = ∫ f_T(E, T) · g(E) dE
```

For the MVP, I(T) can be simplified to a constant or a weak function of T.
Full energy integration is a v0.5+ refinement.

**Step 5: Raw charge density**

```
ρ_eff_raw(x,y) = q · D_it_eff(x,y) · I(T)
```

**Step 6: Mean subtraction (Boussinesq step)**

```
δρ_eff(x,y) = ρ_eff_raw(x,y) − ⟨ρ_eff_raw⟩_spatial
```

This ensures ∫∫ δρ_eff dA ≈ 0. Mean subtraction is not primarily a gauge fix. It defines δρ_eff as a perturbation around a reference/background charge state, suppresses arbitrary uniform charge offsets, and supports charge-neutral solvability for Neumann/periodic Poisson test cases. The electrostatic potential φ may still require a reference/gauge condition.

**Step 7: Poisson equation**

```
∇·(ε∇φ) = −δρ_eff
```

(∇²φ = −δρ_eff/ε is the homogeneous-permittivity simplification.)

### Surface vs Volume Charge Convention

Option C (based on D_it) naturally produces an areal charge density σ_eff [C/m²]. A volumetric Poisson source ρ_eff [C/m³] requires an explicit effective thickness t_eff:

```
δρ_eff = δσ_eff / t_eff
```

- **t_eff** must be demonstrative/T3 calibration-required unless physically justified.
- v0.4 implementation must choose one convention explicitly.
- Tests must verify dimensional consistency.
- No calibrated physical interpretation is allowed.

## C — Levels of Simplification

| Level | Name | Equation | v0.4? |
|-------|------|----------|-------|
| 0 | No implementation | Document only | Current state |
| 1 | Minimal | δρ_eff = q·N_ref·[h(C_def) − ⟨h(C_def)⟩] | ✅ Candidate |
| 2 | Boussinesq-inspired | δρ_eff = q·N_ref·{[1+λTθ+λC·h(C_def)] − mean} | ✅ **Preferred** |
| 3 | Trap occupancy | δρ_eff = q·D_it_ref·{I(T)·h(C_def) − mean[I(T)·h(C_def)]} | 🔜 v0.5+ |

**Level 1** ignores temperature dependence entirely. Simpler but misses
the thermo-defect coupling that is the project's core motivation.

**Level 2** captures both T and C_def contributions through the
Boussinesq-like linearization. This is the recommended v0.4 target.

**Level 3** introduces energy-resolved trap occupancy. Requires Fermi level
knowledge and is deferred to v0.5+.

## Decision

> **Option C-B** is recommended as the preferred closure for ADR-007.
>
> It is the only option that:
> 1. Uses perturbation form (Boussinesq philosophy).
> 2. Couples both T and C_def.
> 3. Handles reference background and neutrality by construction.
> 4. Is testable with manufactured solutions.
> 5. Extends naturally to trap occupancy models (v0.5+).

> [!WARNING]
> **All parameters (D_it_ref, N_ref, λT, λC) are uncalibrated.** They must
> be classified as T0 (demonstrative) or T3 (calibration-required) in the
> parameter registry. No output may be presented as a physical prediction
> of charge density or electrostatic potential in a real device.

> [!IMPORTANT]
> This decision brief **does not authorize implementation.** Implementation
> requires ADR-007 to be promoted from Proposed to Accepted, and
> policy.json to be updated to current_stage v0.4, in a dedicated PR.
