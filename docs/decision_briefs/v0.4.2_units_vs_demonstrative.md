# Decision Brief — v0.4.2 Units vs Demonstrative Mode

> **Date:** 2026-05-11
> **Authors:** Research Council (AI-assisted)
> **ADR:** ADR-008 (Proposed)
> **Scope:** Evaluate three options for SI unit introduction

---

## Background

The v0.4.0 Poisson bridge operates in fully demonstrative mode:
- ε = 1.0 (not physical permittivity)
- N_ref = 1.0 (not physical charge scale)
- t_eff = 1.0 (not physical thickness)
- Domain [0, 1] × [0, 1] (normalized, no physical length)
- φ output is numerically demonstrative, not in volts

The v0.4.1 parameter registry identified T2 candidates (ε_r, D_it) that
could be introduced with SI values, but doing so changes the numerical
scale of every downstream quantity.

This brief compares three options for the transition from demonstrative
to literature-scaled mode.

---

## Option A — Remain Fully Demonstrative

### Description

- Maintain ε = 1.0.
- Maintain N_ref = 1.0.
- Maintain t_eff = 1.0.
- Domain remains [0, 1] × [0, 1] normalized.
- No SI unit conversion introduced.
- All outputs remain demonstrative.

### Advantages

| # | Advantage |
|---|-----------|
| A1 | **Lowest risk of false calibration.** No physical-looking numbers. |
| A2 | **No solver changes required.** Current Jacobi solver is validated. |
| A3 | **Simplest governance.** No new overclaim prevention gates. |
| A4 | **Clear to reviewers.** Obviously demonstrative. |

### Disadvantages

| # | Disadvantage |
|---|-------------|
| A5 | **No physical scale validation.** Cannot verify order-of-magnitude. |
| A6 | **Limited scientific value.** Cannot compare with literature. |
| A7 | **Perpetuates toy model perception.** External reviewers see ε = 1. |

### Risk Level: **Low**

---

## Option B — Literature-Scaled Constants Only

### Description

- Introduce ε = ε_r(Si) · ε₀ = 11.7 × 8.854×10⁻¹² ≈ 1.036×10⁻¹⁰ F/m.
- Maintain N_ref and D_it as demonstrative (not converted to SI).
- Domain may continue normalized or receive explicit unit annotation.
- φ output changes magnitude but is labeled "literature-scaled, not calibrated."

### What Changes

| Parameter | Before | After |
|-----------|--------|-------|
| ε | 1.0 | 1.036×10⁻¹⁰ F/m |
| N_ref | 1.0 | 1.0 (unchanged) |
| D_it | not used | not used |
| t_eff | 1.0 | 1.0 (unchanged) |
| Domain | [0,1]² | [0,1]² or annotated |

### Advantages

| # | Advantage |
|---|-----------|
| B1 | **Physical permittivity.** Correct material property in Poisson. |
| B2 | **Moderate advancement.** Moves toward physical scale. |
| B3 | **Safe boundary.** D_it/N_ref/t_eff remain demonstrative. |
| B4 | **Testable.** Can verify φ scale changes are consistent. |

### Disadvantages

| # | Disadvantage |
|---|-------------|
| B5 | **ε change is dramatic.** φ magnitude changes by ~10¹⁰. |
| B6 | **Partial physical mode.** ε is real but ρ_eff is still demonstrative. |
| B7 | **Risk of misinterpretation.** "Physical ε but toy ρ_eff" is confusing. |
| B8 | **Requires clear mode labeling.** Extra documentation burden. |

### Risk Level: **Medium**

### Implementation Prerequisites

1. ADR-008 must be Accepted.
2. Separate mode flag: `scale_mode = "demonstrative" | "literature_scaled"`.
3. Default mode must remain demonstrative.
4. Unit tests for φ magnitude change.
5. Overclaim gates updated.

---

## Option C — Literature-Scaled Charge Closure

### Description

- Introduce D_it in SI units (J⁻¹ m⁻²).
- Compute σ_eff = q · D_it_SI · ΔE [C/m²] with physical values.
- Introduce t_eff in meters (literature value, e.g., 1 nm).
- Compute ρ_eff = σ_eff / t_eff [C/m³].
- Use ε = ε_r · ε₀ in Poisson.
- Domain may be physical (Lx, Ly in nm).
- Full dimensional chain active.

### What Changes

| Parameter | Before | After |
|-----------|--------|-------|
| ε | 1.0 | ε_r · ε₀ |
| D_it | not used | 10¹⁰–10¹² eV⁻¹cm⁻² → J⁻¹m⁻² |
| σ_eff | not implemented | q · D_it_SI · ΔE |
| t_eff | 1.0 | 0.5–5 nm |
| ρ_eff | demonstrative | σ_eff / t_eff |
| Domain | [0,1]² | physical or annotated |
| φ | demonstrative | literature-scaled [V] |

### Advantages

| # | Advantage |
|---|-----------|
| C1 | **Full dimensional chain.** End-to-end SI units. |
| C2 | **Highest scientific value.** Can compare φ with literature. |
| C3 | **Validates the entire pipeline.** Tests every conversion. |

### Disadvantages

| # | Disadvantage |
|---|-------------|
| C4 | **Highest false calibration risk.** Physical-looking φ without calibration. |
| C5 | **t_eff is arbitrary.** No device-specific justification. |
| C6 | **Energy integration convention required.** ΔE for D_it is non-trivial. |
| C7 | **Domain must be physical or explicitly normalized.** Major design decision. |
| C8 | **Solver numerical range.** ρ_eff and ε differ by 20+ orders of magnitude. |
| C9 | **Overclaim risk is critical.** External readers may treat φ as prediction. |

### Risk Level: **High**

### Implementation Prerequisites

1. ADR-008 must be Accepted.
2. All Option B prerequisites.
3. t_eff justified from literature (not arbitrary).
4. Energy integration convention (ΔE) defined.
5. Domain geometry decision (physical vs normalized with scale annotation).
6. Numerical conditioning tests for solver with ε ~ 10⁻¹⁰.
7. Order-of-magnitude tests for φ.
8. Overclaim gates strengthened.
9. New quality gate: dimensional consistency.
10. New quality gate: φ magnitude sanity check.

## Recommendation

### Preliminary recommendation: **Option B** as the next implementation step

**Rationale:**

1. Option B provides meaningful physical grounding (correct ε_r · ε₀) without
   introducing the full complexity and risk of the charge conversion chain.

2. Option B is reversible — the demonstrative mode is preserved as default.

3. Option B creates a natural checkpoint: if the solver handles ε ~ 10⁻¹⁰
   numerically, the pipeline is ready for Option C. If not, numerical issues
   are identified early.

4. Option B avoids the most dangerous overclaim vectors: D_it conversion,
   t_eff justification, and energy integration conventions.

### Option C should be deferred until:

- Unit infrastructure from Option B is validated.
- t_eff is justified from literature (not arbitrary 1 nm).
- Energy integration convention (ΔE) is formally defined.
- Domain geometry decision is made (physical vs normalized).
- Overclaim gates are updated.
- A separate ADR (ADR-009 or later) authorizes Option C scope.

### Option A is acceptable as a conservative fallback if:

- The project does not need physical scale validation.
- External reviewers do not require physical-looking outputs.
- The focus remains on numerical method verification only.
