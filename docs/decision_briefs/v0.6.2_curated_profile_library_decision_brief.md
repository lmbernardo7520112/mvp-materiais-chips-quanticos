# v0.6.2 Decision Brief — Curated E1/E2 D_it(E) Profile Library

## Context

v0.6.0 implemented the piecewise D_it(E) energy integration API
but shipped no curated profiles. Users must construct their own
PiecewiseDitProfile from scratch.

v0.6.1 re-anchored the roadmap toward a process-to-device
demonstrator for Si/CMOS qubits (ADR-011).

## Problem

Without curated profiles, the energy-integration layer cannot
demonstrate its intended use case: feeding literature-informed
D_it(E) distributions into the C1 chain (N_it → σ_eff).

## Decision

Create a curated E1/E2 D_it(E) profile library providing:

1. Three E1 literature-informed Si/SiO₂ profiles (nominal, high, low).
2. An E2 experimental-prior factory requiring complete metadata.
3. Factories with explicit eV/cm² → J/m² unit conversion.

## Why This Is Not Calibration

- All profiles carry `calibration_status = "not_calibrated"`.
- Profiles are scale approximations from literature ranges.
- No measured energy-resolved curve is used.
- No profile is matched to any specific device.

## Why This Does Not Authorize C2

- C2 (σ_eff → ρ_eff) requires t_eff, which remains blocked.
- ρ_eff requires its own ADR.
- These profiles are input to C1 only.

## Literature Basis

From `docs/literature_review/v0.5.2_c1_literature_scale_benchmark.md`:

| Scale | D_it (eV⁻¹·cm⁻²) | Source |
|-------|-------------------|--------|
| Low | 1×10¹⁰ | Sze & Ng (2006) |
| Nominal | 5×10¹⁰ | Multiple MOS QD refs |
| High | 5×10¹¹ | Degraded interfaces |

## Status

Approved — implemented in v0.6.2.
