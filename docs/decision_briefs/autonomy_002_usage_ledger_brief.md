# Autonomy-002 Usage Ledger Decision Brief

## Decision

Add usage ledger and cost accounting as a required governance layer before any long-running or semi-autonomous execution.

## What It Provides

* event accounting;
* cost estimate accounting;
* retry accounting;
* CI watch accounting;
* artifact accounting;
* approval accounting.

## What It Does Not Provide

* exact Antigravity token metering;
* billing integration;
* paid API access;
* SDK runtime;
* autonomous spending.

## Rationale

BudgetOps prevents uncontrolled spending. Usage Ledger records execution facts and estimates, enabling post-run accountability.
