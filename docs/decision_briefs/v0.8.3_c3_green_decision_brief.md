# v0.8.3 C3 Green Decision Brief

## Context
During the execution of v0.8.3 — C3 GREEN mínimo, the module `c3_solver_projection.py` was created to satisfy the C3-A RED tests. Three RED sentinel tests (`test_c3_projection_module_imports`, `test_no_poisson_solver_import_in_projection_module`, `test_no_ai_for_science_imports`) were expecting `ImportError` because the module did not exist.

## Decision
Three RED sentinel tests that previously expected ImportError were converted into GREEN isolation tests after the authorized creation of c3_solver_projection.py. This does not weaken the test suite; it preserves the original guardrails by checking successful import plus absence of solver, Poisson runtime, ML imports, physical phi and AI-for-Science contamination.

## Consequences
- The RED tests correctly fail if isolation rules are broken.
- The full test suite runs successfully with `c3_solver_projection.py` implemented.
- We maintain the strict classical vs AI-for-Science separation.
