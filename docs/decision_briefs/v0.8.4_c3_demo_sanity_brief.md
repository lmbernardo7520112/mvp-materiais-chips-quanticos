# Decision Brief: v0.8.4 C3 Demo Sanity Checks

## Context
Phase v0.8.3 established the core of C3 projection (mapping C2 continuous source defects onto a grid representation), but there was no explicit demonstration pipeline to validate charge conservation and integration with `generate_all_results.py`.

## Decision
Create a strict, isolated demo script (`scripts/run_c3_projection_demo.py`) to systematically demonstrate C3 grid mapping. This demo must not import `poisson_solver_2d`, to enforce the separation of C3 source mapping from C3 potential solving, adhering strictly to ADR-013.

## Rationale
Testing in isolation proves that C3's mapping functions correctly preserve the physical contracts (charge conservation, sign, metadata) before adding the complexity of solving the Poisson equation.

## Consequences
- Validation passes perfectly showing `O(1e-25)` numerical precision on charge conservation.
- C3 projection core is proven ready for solver coupling in a future release.
