# v0.7.0 C2 Interface Source / Regularization Decision Brief

## Core Question

How should σ_eff [C/m²] be represented for future electrostatic coupling?

## Not the Right Question

> "What t_eff should we use?"

This question is rejected because:

- It presupposes volume mapping as the only path.
- It invites selection of an arbitrary thickness parameter.
- It conflates numerical regularization with physical thickness.
- No universal t_eff exists for Si/SiO₂ interface traps.

## Correct Question

> "Should σ_eff remain a sheet/interface source, or be mapped to a
> conservative regularized volume source?"

## Option Scoring

| Option | Physical fidelity | Overclaim risk | Numerical practicality | Roadmap alignment | Weighted Score |
|--------|:-:|:-:|:-:|:-:|:-:|
| C2-A Interface sheet source | 9.5 | 2.0 | 7.0 | 9.0 | 91 |
| C2-B Conservative volume regularization | 7.0 | 5.0 | 8.5 | 8.0 | 80 |
| C2-C Depth prior for sensitivity | 8.0 | 6.0 | 7.0 | 7.5 | 76 |
| C2-D Generic physical t_eff | 4.0 | 8.5 | 6.0 | 4.0 | 48 |
| C2-E Calibrated t_eff | 9.0 theoretical | 9.5 now | 3.0 | 2.0 | 35 |

**Scoring notes:**

- Physical fidelity: how closely the formulation matches the physics.
- Overclaim risk: probability of false calibration/interpretation.
- Numerical practicality: ease of implementation in future solver.
- Roadmap alignment: compatibility with process-to-device demonstrator.
- Weighted score: (fidelity×3 + (10−risk)×3 + practicality×2 + alignment×2) / 10.

## Recommendation

| Option | Recommendation |
|--------|---------------|
| C2-A | **Primary** — interface sheet source / jump condition |
| C2-B | **Fallback** — conservative volume regularization with l_reg |
| C2-C | **Conditional** — depth prior for sensitivity studies only |
| C2-D | **Rejected** — generic physical t_eff not accepted |
| C2-E | **Blocked** — calibrated t_eff requires future ADR |

## Why This Is Efficient

1. **Does not paralyze the roadmap.** ADR-012 defines the strategy
   without requiring implementation. Future RED/GREEN can proceed
   immediately after acceptance.

2. **Avoids inventing t_eff.** The primary path (C2-A) does not
   require any thickness parameter. The fallback (C2-B) uses l_reg
   as an explicit numerical regularization, not a physical claim.

3. **Prepares future RED.** The future RED plan (v0.7.0) documents
   15 candidate tests for C2 implementation.

4. **Protects future electrostatic coupling.** By choosing sheet
   source as primary, the solver integration (v0.8.0) inherits a
   physically faithful interface representation.

5. **Preserves scientific credibility.** No calibration claims, no
   device prediction, no coherence/fidelity assertion.

## What This ADR Does NOT Do

- Does not implement any code.
- Does not create tests.
- Does not authorize ρ_eff computation.
- Does not authorize solver coupling.
- Does not accept ADR-012 (it remains Proposed).
