# v0.5.2 C1 Scale Positioning Brief

## Core Question

Where do the σ_eff values produced by C1 surface-density bookkeeping
sit relative to plausible literature ranges for D_it, ΔE_window, and
occupancy in silicon spin-qubit interfaces?

## Scale Classes

| Class | D_it Range (eV⁻¹·cm⁻²) | Meaning |
|-------|-------------------------|---------|
| toy_low | < 10⁹ | Below any physical relevance; pure numerical test |
| literature_plausible_low | 10¹⁰ | Best-quality passivated Si/SiO₂ |
| literature_plausible_nominal | 5×10¹⁰ | Typical research-grade MOS QD interface |
| literature_plausible_high | 5×10¹¹ | Upper bound for degraded/unpassivated interfaces |
| aggressive_upper_bound | 10¹² | Stress test; rarely physical for modern devices |

## Decision

### Standard benchmark scenarios for future C1 demos

Use `low_clean`, `nominal`, and `high_degraded` as the three-point
benchmark set for all future C1 demonstration artifacts.

### Stress test scenarios

Use `aggressive` and `aggressive_neg` only for boundary testing.
These values should NEVER be presented as physically representative.

### Values NOT to be used in physical claims

- Any σ_eff value from this benchmark is NOT calibrated.
- No scenario should be cited as "predicting" device behavior.
- The `aggressive_upper_bound` class exists solely for numerical
  robustness testing.

### What still impedes C2

1. No ADR authorizing ρ_eff = σ_eff / t_eff conversion.
2. No ADR authorizing solver coupling (Poisson source term).
3. No regularization strategy for t_eff → 0 singularity.
4. No experimental data for validation.
5. No physical interpretation of φ output.
