# Decision Brief — v0.2.1 vs v0.3

> **Data:** 2026-05-07  
> **Autor:** Staff Research Software Engineer (AI-assisted)  
> **Status:** APROVADO — v0.2.1 institucional antes de v0.3

---

## 1. Contexto da Decisão

A v0.2.0 fechou o marco de robustez numérica 2D. O roadmap ADR-005
estabelece v0.3 como "defect-like reaction-diffusion". A questão é:

> **Avançar diretamente para v0.3 ou inserir uma v0.2.1 institucional?**

---

## 2. Decisão

**Inserir v0.2.1 antes de v0.3.**

A v0.2.1 é uma release institucional e de reprodutibilidade, sem física
nova. A v0.3 será especificada e implementada depois, com ADR própria.

---

## 3. Justificativa

A v0.2.0 criou um marco técnico significativo, mas ele não está
empacotado para comunicação institucional:

- Não havia release notes formais
- Não havia tutorial de reprodução
- Não havia notebook demonstrativo
- Não havia narrativa legível para parceiros potenciais
- Não havia decision brief documentando o caminho estratégico

Publicar v0.3 sem essa camada institucional desperdiçaria o valor
comunicável do marco v0.2.0.

---

## 4. Respostas às Perguntas de Banca

### 4.1 Qual problema físico será modelado em v0.3?

A v0.3 deve introduzir uma variável do tipo **defect-like** com dinâmica
de **reaction-diffusion**:

```
∂C_def/∂t = ∇·(D(T)∇C_def) + G(T) − R(T, C_def)
```

Onde:
- `C_def(x,y,t)` é uma variável adimensional representando densidade
  normalizada de defeitos térmicos (vacâncias, intersticiais, armadilhas)
- `G(T)` é taxa de geração dependente de temperatura
- `R(T, C_def)` é taxa de recombinação/relaxação
- `D(T)` é difusividade tipo Arrhenius (já implementada em 1D)

O problema é **modelar a distribuição espacial de heterogeneidades
de origem térmica** que, em etapas posteriores (v0.4+), serão
convertidas em carga efetiva para eletrostática.

### 4.2 Qual variável será proxy e qual fisicamente interpretável?

| Variável | Natureza na v0.3 | Evolução futura |
|----------|-------------------|-----------------|
| T(x,y,t) | Físico aproximado (α do Si) | Mantém |
| C_def(x,y,t) | **Proxy adimensional** | v0.5+: calibração |
| G(T) | Toy/demonstrativo | v0.5+: literatura |
| R(T, C_def) | Toy/demonstrativo | v0.5+: literatura |
| D(T) | Arrhenius demonstrativo | v0.5+: calibração |

**Decisão explícita:** `C_def` permanece adimensional na v0.3.
Não será apresentado como concentração de defeitos calibrada.

### 4.3 Quais parâmetros terão origem em literatura?

Na v0.3, **nenhum parâmetro novo será calibrado**. Os termos G(T) e
R(T, C_def) usarão formas funcionais fisicamente motivadas (e.g.,
Arrhenius para G(T), relaxação linear para R), mas com valores
toy/demonstrativos.

A calibração com literatura (energias de formação de vacâncias em Si,
taxas de recombinação de intersticiais, dados de Sinno et al. 1998)
será deferida para v0.5+ e documentada como technical debt.

### 4.4 Como v0.3 se conecta a Poisson v0.4?

A conexão formal é:

```
C_def(x,y) → ρ_eff(x,y) = q_eff · C_def(x,y)
```

Onde `q_eff` é um fator de acoplamento que converte a variável de
defeito em densidade de carga efetiva. Na v0.4, `ρ_eff` será o
termo fonte da equação de Poisson (forma geral):

```
∇·(ε∇φ(x,y)) = −ρ_eff(x,y)
```

(Nota: ∇²φ = −ρ_eff/ε é o caso homogêneo simplificado, válido quando ε
é constante. Para o roadmap v0.4, a forma geral com permissividade
espacialmente dependente é a referência correta, especialmente em
cenários semicondutor/óxido/interface.)

A v0.3 deve produzir `C_def(x,y)` final (após freeze-in ou steady-state)
que sirva como input direto para v0.4. Isso implica:

- v0.3 deve exportar `C_def_final` como array
- v0.3 deve gerar figura de `C_def_final(x,y)`
- v0.3 não precisa resolver Poisson

### 4.5 Quais resultados ainda não podem ser inferidos?

Após v0.3, **ainda não** será possível inferir:

- Potencial eletrostático V(x,y)
- Níveis de energia de confinamento
- Coerência ou decoerência de qubits
- Performance de portas quânticas
- Comparação com TCAD comercial
- Comportamento de dispositivo real
- Parâmetros de processo industrial

O que **poderá** ser inferido:

- Distribuição espacial de heterogeneidades de origem térmica
- Sensibilidade da distribuição a parâmetros de processo idealizado
- Convergência numérica da equação de reaction-diffusion
- Qualidade da infraestrutura computacional

### 4.6 Qual artefato institucional será enviado a parceiros?

O artefato institucional é o **resumo CNPEM/LNNano** criado na v0.2.1,
acompanhado de:

- Link para o repositório (privado, com acesso controlado)
- Release notes formais
- Tutorial de reprodução
- Notebook demonstrativo executável
- Roadmap visual (ADR-005)
- Escopo de limitações explícito

O tom deve ser:

> "Estamos construindo infraestrutura computacional para modelagem
> process-to-device de materiais semicondutores. Gostaríamos de
> avaliação técnica e indicação de possíveis orientadores ou
> colaboradores na área de defeitos em silício e dispositivos quânticos."

---

## 5. Pré-Condições para v0.3

A v0.3 NÃO deve começar sem:

1. **ADR-006** definindo formalmente C_def, G(T), R(T, C_def)
2. **v0.3 implementation plan** com MUST/SHOULD/WON'T
3. **v0.3 acceptance gates** com testes de convergência
4. **Decisão explícita** sobre se C_def é 1D ou 2D
5. **Deliberação** sobre acoplamento térmico-difusivo (one-way ou two-way)

```
v0.2.0 ✅  Marco técnico: robustez numérica 2D
v0.2.1 ✅  Marco institucional: reprodução, comunicação, decisão
v0.3.0 🔜  Marco científico: defect-like reaction-diffusion
           (requer ADR-006, spec, gates — NÃO inicia sem deliberação)
```

## 7. Notas Finais

- A distinção entre v0.2.1 e v0.3 é **deliberada e defensável**
- v0.2.1 transforma engenharia em comunicação
- v0.3 transforma infraestrutura em física
- Pular v0.2.1 desperdiçaria o valor comunicável da v0.2.0
- Pular v0.3 estagnaria o projeto no "substrato metodológico"
